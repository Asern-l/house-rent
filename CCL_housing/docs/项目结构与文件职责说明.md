# 项目结构与文件职责说明

## 文档目的
- 说明每个文件夹对应模块。

- 作为团队协作和后续重构的统一索引。

## 更新规则（必须执行）
1. 新增文件：提交代码时必须同步在本文档增加该文件作用。
2. 删除文件：删除代码时必须同步在本文档删除对应说明。
3. 文件改职责：如果文件用途发生变化，必须同步更新说明。
4. 自动生成目录（如 `node_modules`、`dist`、`data/database.sqlite`）不在此文档登记。

## 3. 前端模块（`apps/frontend`）
### 目录重组说明（已执行）
- `src/app`：应用壳层目录，承载全局路由、整体布局、应用级状态入口。
- `src/app/providers`：应用级 Provider 目录，集中管理鉴权、会话、钱包等全局上下文。
- `src/pages`：页面目录，按路由拆分业务页面组件。
- `src/shared/api`：通用 API 封装目录，统一请求入口与鉴权头处理。
- `src/shared/blockchain`：区块链共享资源目录，存放 ABI 等链交互基础资源。
- `src/index.css`：全局样式入口文件。
- `src/main.jsx`：前端渲染入口文件，负责挂载路由与全局 Provider。
