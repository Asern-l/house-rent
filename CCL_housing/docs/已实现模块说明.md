# 已实现模块说明

## 文档目的
- 记录当前重构版项目（`CCL_housing`）已经完成并可用的功能模块。
- 便于后续开发、交接和答辩时快速说明项目现状。

## 1. 用户与鉴权模块
### 实现内容
- 用户注册：支持 `landlord` / `tenant` 角色。
- 用户登录：返回 JWT 令牌。
- 用户信息查询：`GET /auth/me`。
- 用户信息更新：`PUT /auth/me`，支持昵称更新与钱包强绑定策略（首次绑定后不可更改/解绑，仅支持重连同一地址）。

### 主要位置
- `apps/backend/src/routes/auth.js`
- `apps/backend/src/auth.js`

## 2. 房源模块
### 实现内容
- 房东发布房源：标题、地址、租金、户型等基础字段。
- 房源列表查询：支持按关键字与状态查询。
- 房源详情查询。
- 我的房源查询（房东）。
- 房源状态机：`available`、`locked`、`rented`、`offline`、`closed`。
- 房源上下架约束：仅允许 `available <-> offline`，流程中状态（`locked/rented/closed`）禁止手动改动。

### 主要位置
- `apps/backend/src/routes/listings.js`
- 前端页面：`apps/frontend/src/pages/ListingsPage.jsx`、`ListingDetail.jsx`、`MyListings.jsx`、`PublishListing.jsx`

## 3. 合同签署模块
### 实现内容
- 租客发起合同：自动生成合同内容 JSON 和 `content_hash`。
- 并发保护：创建合同时原子抢锁房源（`available -> locked`），防止双租客并发申请。
- 租客签署：原子条件更新 `pending -> tenant_signed`。
- 房东签署：原子条件更新 `tenant_signed -> pending_payment`。
- 首笔一次性支付成功：`pending_payment -> active`，并同步房源 `locked -> rented`。
- 首笔一次性支付链路：租客调用合约 `recordRentPayment(payable)`，合约转付房东后，后端回写支付记录并激活合同。
- 合同取消：签署阶段允许取消，并在满足条件时释放房源锁定状态。
- 上链回写：记录交易哈希 `txHash`。
- 上链回写幂等约束：同一合同仅允许首次回写 `txHash`。
- 支付回写：当前仅支持 `initial`（一次性支付）写入，且仅允许一次并具备幂等约束。
- 客户端失败上报：签署异常可通过接口写入日志。

### 主要位置
- `apps/backend/src/routes/contracts.js`
- 前端页面：`apps/frontend/src/pages/ContractPage.jsx`

### 主要位置
