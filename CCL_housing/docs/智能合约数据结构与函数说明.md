# 智能合约数据结构与函数说明

## 文档范围
- 合约文件：`CCL_housing/blockchain/contracts/RentalChain.sol`
- 说明内容：枚举、结构体、状态变量、事件、函数与权限

## 当前阶段实现范围说明
- 当前项目以原型可运行为目标，合约侧**暂时先实现并优先联调**以下两部分：
  1. 房源存证部分（`storeListing` / `getListing` / `getListingCount`）
  2. 合同存证部分（`storeContract` / `getContract`）
- 押金托管、纠纷仲裁能力虽然在合约中已定义，但系统侧暂未作为当前阶段必做闭环功能。
- 租金支付当前已接入“租客直连合约支付 + 合约转付房东”的最小闭环。

## 1. 枚举定义

### 1.1 `DepositStatus`
- `None`
- `Paid`
- `Refunding`
- `Refunded`
- `PartiallyRefunded`
- `Disputed`

用途：表示押金托管与退款流程状态。

## 2. 结构体定义

### 2.1 `ListingRecord`
字段：
- `string listingId`：房源ID
- `address landlord`：房东地址
- `uint256 createdAt`：创建时间戳
- `bool exists`：是否存在

说明：原型阶段已移除 `aiScore` 与 `imageHashes` 字段。

### 2.2 `ContractRecord`
字段：
- `string contractId`：合同ID
- `string listingId`：关联房源ID
- `bytes32 contractHash`：合同内容哈希
- `address tenant`：租客地址
- `address landlord`：房东地址
- `uint256 createdAt`：创建时间戳
- `bool exists`：是否存在

### 2.3 `DepositRecord`
字段：
- `string contractId`：关联合同ID
- `address tenant`：租客地址
- `address landlord`：房东地址
- `uint256 amount`：押金金额
- `DepositStatus status`：押金状态
- `uint256 paidAt`：支付时间戳
- `uint256 deductionAmount`：扣款金额
- `string deductionReason`：扣款理由
- `bool exists`：是否存在

## 6. 函数说明

### 6.1 构造与管理
- `constructor()`：设置 `owner`
- `transferOwnership(address _newOwner)`：转移所有权
- `receive() external payable`：接收 ETH

### 6.2 房源存证
- `storeListing(string _listingId)`
  - 权限：任意外部地址
  - 作用：写入最小房源存证记录
- `getListing(string _listingId)`
  - 返回：`landlord, createdAt, exists`
- `getListingCount()`
  - 返回：房源数量

### 6.3 合同存证
- `storeContract(string _contractId, string _listingId, bytes32 _contractHash, address _tenant, address _landlord)`
- `getContract(string _contractId)`

### 6.4 押金托管
- `payDeposit(string _contractId)` `payable`
- `requestRefund(string _contractId)`
- `confirmRefund(string _contractId)`
- `proposeDeduction(string _contractId, uint256 _amount, string _reason)`
- `acceptDeduction(string _contractId)`
- `disputeDeposit(string _contractId)`
- `resolveDeposit(string _contractId, bool _toTenant)` `onlyOwner`
- `getDeposit(string _contractId)`

### 6.5 租金支付与存证
- `recordRentPayment(string _contractId, address _tenant, address _landlord, uint256 _amount, string _alipayOrderNo)` `payable`
- 权限与行为：
  - 仅租客地址可调用（`msg.sender == _tenant`）
  - 参数租客/房东地址需与合约中该合同绑定地址一致
  - `msg.value == _amount`
  - 合约收到 ETH 后立即转给房东地址，并触发 `RentReceipt` 事件

### 6.6 支付授权签名方案（待实现）
- 方案目标：在不牺牲“用户自发链上支付”的前提下，增加链上准入风控。
- 设计要点：
  - 后端在支付前签发一次性授权：`contractId + payer + amount + deadline + nonce`。
  - 合约端使用验签（`ecrecover` 或 EIP-712）校验授权签名。
  - 仅授权有效、未过期、未使用（nonce 未消费）时允许支付。
- 设计理由：
  - 防止用户绕过前端直接对不允许状态（如 `cancelled`）发起支付。
  - 防重放、防跨合同复用授权、防参数篡改。
  - 保持去中心化支付流程：链上交易仍由用户钱包发起并支付 gas。
