# CCL Housing（信源链租房原型）

## 1. 项目定位
CCL Housing 是一个围绕“租房合同签署与链上存证”的全栈原型项目。  
核心目标是把房源、合同、支付、验真串成可运行闭环，并支持本地链与测试网双环境隔离。

## 2. 核心业务闭环
1. 用户注册与登录（租客/房东角色）。
2. 房东发布房源，租客浏览并发起签约。
3. 租客签署 -> 房东签署 -> 合同状态进入待支付。
4. 链上写入合同存证（哈希上链）。
5. 首笔支付成功后合同生效；当前首笔支付会按 `0.1%` 向平台收取手续费，手续费从房东应收中扣除，租客支付总额不变。  
6. 平台验真模块可查询合同哈希、房源公开快照与状态一致性。
7. 独立 verifier 可脱离平台账号系统，直接读取链上、IPFS 和本地 PDF 做独立验真；合同 PDF 现已支持重算合同哈希、本地自验签，并在结果页显示付款/生效状态与中国时间时间线。
8. 平台已接入站内通知：房源申请、条款协商、签署、待支付、支付确认、超时取消、房源反馈与租后评价等事件会写入服务器通知中心，并在前端显示未读数与通知列表。

## 3. 技术架构
### 3.1 前端（apps/frontend）
- React + Vite + Tailwind。
- 负责页面交互、钱包连接、链上交易触发、调用后端 API。
- 支持 Sepolia / Local 两种网络切换。

### 3.2 后端（apps/backend）
- Node.js + Express + sql.js。
- 负责鉴权、房源、合同、支付、站内通知、平台内验真等业务接口。
- 通过 `CHAIN_ENV` 实现环境隔离：
  - `sepolia` -> `database.sepolia.sqlite`
  - `local` -> `database.local.sqlite`

### 3.3 区块链（blockchain）
- Solidity + Hardhat。
- 合约负责合同哈希存证与支付记录相关链上行为。  
- 首笔支付事件当前会同时记录：
  - 支付总额 `gross`
  - 平台手续费 `platformFeeWei`
  - 房东实收 `landlordNetWei`
  - 平台手续费接收地址 `platformFeeRecipient`
- 部署输出按网络拆分：
  - `deployments-rental-sepolia.json`
  - `deployments-rental-localhost.json`

### 3.4 脚本与文档
- `scripts`：启动、重置、部署、回归等自动化脚本。
- `verifier`：独立验真程序，不依赖平台账号系统，单独提供本地 Web 前端；支持合同 PDF 强校验、房源快照/IPFS 验真与评论联动验真，底层 node 脚本仅作为开发调试能力保留。
- `docs`：结构文档、接口文档、配置文档、待实现清单等。

## 4. 目录结构（详细）
- `apps/backend`：后端代码与本地数据库。
- `apps/frontend`：前端页面、Provider、链交互资源（ABI）。
- `blockchain`：合约源码、Hardhat 配置、部署脚本。
- `scripts`：开发与演示脚本（单环境启动、并行启动、持久化本地链等）。
- `verifier`：独立验真工具目录，负责本地 PDF、链上和 IPFS 联动验真。
- `docs`：项目文档体系。
- `logs`：运行日志输出目录。

## 5. 环境模型
### 5.1 Sepolia（测试网）
- 用于真实测试网交互、交易哈希可在区块浏览器验证。

### 5.2 Local（本地链）
- 用于快速调试、可重置、可持久化节点（Ganache）。
- 与 Sepolia 在数据库、部署地址、会话上隔离。

## 6. 关键设计约束
1. 配置与业务解耦：环境切换通过变量驱动，不复制业务代码。
2. 数据隔离优先：不同网络绝不共用同一业务库。
3. 合同正文继续由平台数据库主存，不上 IPFS；独立合同验真依赖用户本地保存的 PDF。
4. 公开验真材料（房源图片、房源公开快照、房源反馈、租后评价）走 IPFS。
5. 文档同步：结构/配置/脚本变更必须更新 docs。
