# 配置与部署教程

## 文档目的
- 指导项目从本地运行到合约部署的配置流程。
- 采用“合约部署优先”的推荐顺序，减少联调阶段的地址与网络不一致问题。

## 1. 环境准备
1. 安装 Node.js 18+。
2. 准备 MetaMask 钱包（用于前端真实交易）。
3. 准备 Sepolia 测试网 ETH（用于部署与交易 gas）。

## 2. 合约部署优先（推荐流程）

### 2.1 安装依赖
在项目根目录执行：

```bash
npm --prefix apps/backend install
npm --prefix apps/frontend install
npm --prefix blockchain install
```

### 2.2 配置 `blockchain/.env`
参考 `blockchain/.env.example`，创建 `blockchain/.env`：

```env
SEPOLIA_RPC_URL=https://ethereum-sepolia.publicnode.com
PRIVATE_KEY=0x你的部署私钥
```

注意：
- `PRIVATE_KEY` 仅用于部署或服务端代签名场景。
- 日常前端使用 MetaMask，不需要把私钥放到前端项目。

### 2.3 编译与部署
```bash
npm --prefix blockchain run compile
npm --prefix blockchain run deploy:sepolia
```

部署成功后会输出合约地址，并更新部署文件（如 `blockchain/deployments-rental-sepolia.json`）。

### 2.4 回填前端合约地址
在 `apps/frontend/.env` 中设置：

```env
VITE_CONTRACT_ADDRESS=0x部署输出的合约地址
```

说明：
- 当前前端已移除本地默认地址兜底，`VITE_CONTRACT_ADDRESS` 未配置或格式错误时会直接禁止上链操作。

### 2.5 配置后端 `apps/backend/.env`
参考 `apps/backend/.env.example`，创建 `apps/backend/.env`：

```env
JWT_SECRET=change_me_to_a_long_random_string
PORT=3000
```

### 2.6 启动服务
```bash
npm --prefix apps/backend run dev
npm --prefix apps/frontend run dev
```

或使用一键脚本：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-basic-services.ps1
```

### 2.7 健康检查
- 后端：`http://localhost:3000/api/health`
- 前端：按 Vite 输出地址访问（通常 `http://localhost:5173`）

## 3. 本地快速启动（不部署合约）
当你只想验证后端与前端基础流程时：

1. 安装依赖（后端 + 前端）。
2. 配置 `apps/backend/.env`（`JWT_SECRET`、`PORT`）。
3. 启动前后端服务。

说明：
- 该模式不要求配置 `blockchain/.env`。
- 若涉及前端真实链上交互，仍需配置 `VITE_CONTRACT_ADDRESS` 并连接 MetaMask 到 Sepolia。

## 4. 回归脚本（无需私钥）
执行：

```bash
node scripts/test_contract_sign_flow.js
```

说明：
- 该脚本会使用测试钱包地址做业务流验证。
- 不依赖 `blockchain/.env` 中私钥。

## 5. 端口联动说明
- 默认后端端口是 `3000`。
- 如修改后端端口，请同步检查前端代理配置：`apps/frontend/vite.config.js` 中 `/api` 代理目标。
- 两者不一致时，前端会出现接口超时或代理错误。

## 6. 时间口径说明（北京时间）
- 当前后端数据库写入时间口径已统一为北京时间（`UTC+8`）。
- 典型字段包括：`created_at`、`updated_at`、`paid_at`、`tenant_signed_at`、`landlord_signed_at`、`expires_at`。
- 若你已同意重置数据库，请删除 `apps/backend/data/database.sqlite` 后重启后端，让新结构与新时间口径完全生效。

## 7. 常见问题
1. `vite is not recognized`
- 前端依赖未安装，执行：`npm --prefix apps/frontend install`。

2. `Cannot find module 'dotenv'`
- 后端依赖未安装，执行：`npm --prefix apps/backend install`。

3. MetaMask 提示网络不支持
- 切换到 Sepolia 网络。

4. 合约地址错误
- 检查 `apps/frontend/.env` 的 `VITE_CONTRACT_ADDRESS` 是否最新。

5. 部署失败（余额不足）
- 给部署钱包补充 Sepolia 测试币。

6. 签署或发布失败
- 查看日志：`logs/sign-flow-error.log`、`logs/listing-error.log`。
