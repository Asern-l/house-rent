# 配置与部署教程（完整）

## 1. 文档范围
本文档只讲“如何配置与运行”，不讲项目结构设计。  
包含：环境变量、MetaMask 配置、Sepolia 部署、Local 持久化节点、双环境并行运行、IPFS 本地节点、独立 verifier 复现。

## 2. 前置准备
1. 安装 Node.js 18+。
2. 安装 MetaMask 浏览器插件。
3. 项目根目录执行依赖安装：
```bash
npm --prefix apps/backend install
npm --prefix apps/frontend install
npm --prefix blockchain install
cd verifier && npm install
```

## 3. 环境变量配置

### 3.1 后端配置（apps/backend/.env）
参考 `apps/backend/.env.example`：
```env
JWT_SECRET=change_me_to_a_long_random_string
PORT=3000
HOST=127.0.0.1
AUTH_PORT=3005
CHAIN_ENV=sepolia
JSON_BODY_LIMIT=100mb
PAYMENT_WINDOW_HOURS=2
```
说明：
- `CHAIN_ENV` 可选 `sepolia` / `local`。
- `PORT` 仅影响当前后端实例监听端口。
- `HOST` 默认 `127.0.0.1`，仅监听本机回环地址（不再默认绑定 `0.0.0.0`）。
- `AUTH_PORT` 为独立登录服务端口（默认 `3005`）。
- `JSON_BODY_LIMIT` 为请求体上限（默认 `100mb`），用于多图 base64 上传；若触发 `413 request entity too large`，可按机器容量适当调大。
- `PAYMENT_WINDOW_HOURS` 为合同首笔支付窗口；后端超时取消逻辑与链上部署参数都应保持一致。
- 若使用 `scripts/ps1/start-sepolia-services.ps1` / `scripts/ps1/start-local-services.ps1` / `scripts/ps1/start-parallel-services.ps1` 启动：不要手动修改 `CHAIN_ENV`；脚本会注入并覆盖当前进程的 `CHAIN_ENV`。
- 账号数据按环境拆分：`apps/backend/data/users.sepolia.sqlite` 与 `apps/backend/data/users.local.sqlite`。
- 认证服务按登录目标网络选择账号库；业务后端按自身 `CHAIN_ENV` 读取对应账号库。
- 房源图片上传目录：`apps/backend/data/uploads/`（运行时数据目录，不提交 Git）。
- 若启用 IPFS 验真材料层，还应补充：
```env
IPFS_ENABLED=1
IPFS_API_URL=http://127.0.0.1:5001/api/v0
IPFS_GATEWAY_URL=http://127.0.0.1:8080/ipfs
```

### 3.2 前端配置（apps/frontend/.env）
参考 `apps/frontend/.env.example`：
```env
VITE_DEFAULT_NETWORK=sepolia
VITE_API_BASE_AUTH=/api-auth
VITE_API_BASE_SEPOLIA=/api
VITE_API_BASE_LOCAL=/api-local
VITE_CONTRACT_ADDRESS_SEPOLIA=0xYourSepoliaContractAddress
VITE_CONTRACT_ADDRESS_LOCAL=0xYourLocalContractAddress
```
说明：
- 登录/注册/用户资料固定走 `VITE_API_BASE_AUTH`（默认 `/api-auth` -> `127.0.0.1:3005`）。
- 前端会按当前网络读取对应 API 前缀和合约地址。
- 任一网络未配置地址时，该网络的链上操作会被阻断。

### 3.3 区块链部署配置（blockchain/.env）
仅 Sepolia 部署需要：
```env
SEPOLIA_RPC_URL=https://ethereum-sepolia.publicnode.com
PRIVATE_KEY=0x你的部署私钥
PAYMENT_WINDOW_HOURS=2
TRUSTED_SIGNER_ADDRESS=0x平台permit签名地址
PLATFORM_FEE_RECIPIENT_ADDRESS=0x平台手续费接收地址
```
说明：
- `PRIVATE_KEY` 仅用于部署脚本，不用于前端。
- `PAYMENT_WINDOW_HOURS` 会在部署时换算为合约构造参数 `paymentWindowMs`；重部署后才会生效。
- `TRUSTED_SIGNER_ADDRESS` 用于 permit 门禁签名者；未配置时默认回退到部署地址。
- `PLATFORM_FEE_RECIPIENT_ADDRESS` 用于接收首笔支付平台手续费；未配置时默认回退到 `TRUSTED_SIGNER_ADDRESS`。

### 3.4 网络会话隔离（已彻底实现）
- 会话按网络独立存储：`token:sepolia` / `user:sepolia` 与 `token:local` / `user:local`。
- 不再使用全局 `token` 回退。
- 切换网络后若该网络尚未登录，需要在该网络登录一次，这是预期行为（隔离而非串用）。
- `preferredNetwork` 仅表示当前前端选择的网络，不代表跨网络共享会话。

2. 启用测试网络显示。
3. 切换到 `Sepolia`。

1. 打开 MetaMask -> 添加网络 -> 手动添加。
2. 填写：
- 网络名称：`Local EVM (31337)`
- 新 RPC URL：`http://127.0.0.1:8545`
- 链 ID：`31337`
- 货币符号：`ETH`
3. 保存并切换到该网络。

### 4.3 导入本地测试账户（可选）
当使用本地链（当前为 Ganache 节点）时，可把测试私钥导入 MetaMask，用于本地签约和支付演示。  
注意：测试私钥仅限本地测试环境，不可用于真实资产钱包。

补充说明（本地测试转账）：
- 你可以把“当前节点启动日志里显示的测试账户”导入 MetaMask 后，在 `Local EVM (31337)` 网络向你自己的本地测试钱包地址转账测试 ETH。
- 该转账仅发生在本地链，不会进入 Sepolia 或主网，也不涉及真实资产。
- 建议只在 `chainId=31337` 下操作，避免误切换到其他网络。

重要说明（避免过期）：
- 文档不再内置固定“10个私钥/助记词”清单，因为本地节点数据目录变化或重建后，账户会变化。
- 以你本次启动 `scripts/ps1/start-persistent-local-node.ps1` 的终端输出为准（`Available Accounts` / `Private Keys` / `Mnemonic`）。

### 4.4 签约与支付的签名说明
- 合同签署：租客/房东在合同页点击签署时，MetaMask 会弹出“消息签名”。该签名不转账、不消耗 gas，用于证明当前钱包同意合同正文哈希。
- gas 锁仓：租客点击“租客签署”时，还会追加一笔链上交易 `lockGasCompensationEscrow(...)`，把 gas 补偿金额锁进合约。
- 房东签署并上链：房东点击“房东签署并上链”时，会先完成消息签名，再发起同一流程中的链上交易；该交易会同时完成合同存证、按公式报销房东 gas、并把锁仓剩余金额退回租客。
- 房租支付：租客点击支付按钮时，由租客钱包直接向合约发起链上支付交易，不再经过后端 `payments/authorization` 平台代签。
- 续约合同：仅租客可在父合同页发起；续约月数可编辑，起始时间固定为父合同结束时间。若父合同未过期时先完成续约支付，前端会显示“已支付待接续”。
- 链上占用表达：房源链上不再单独维护 `rented`；是否被占用由生效合同映射 `activeContractByListing` 表达，支付成功后合同才会进入链上 `Active`。
- 取消合同：若租客已完成 gas 锁仓但房东尚未签署，房东或租客取消前都会先发起链上统一释放 `cancelPendingGasAuthorization(...)`，合约会把锁仓退款给租客，再执行后端取消。
- 因此完整流程包含“消息签名”和“链上交易确认”两类动作；支付阶段不需要额外配置平台支付私钥。

## 7. Sepolia + Local 并行运行

### 7.1 一键并行启动
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-parallel-services.ps1
```

### 7.2 并行端口
- 登录服务（共享账号）：`http://127.0.0.1:3005`
- 后端（Sepolia）：`http://127.0.0.1:3000`
- 后端（Local）：`http://127.0.0.1:3002`
- 前端（Vite）：默认 `http://127.0.0.1:3001`

### 7.3 前端切换逻辑
- 认证接口（`/api-auth/*`）固定走 3005，不随网络切换。
- 选择 `Sepolia`：请求走 `/api` -> 3000。
- 选择 `Local EVM (31337)`：请求走 `/api-local` -> 3002。

### 7.4 重要说明（避免误解）
- 并行模式下虽然只有一套代码，但会启动两个后端进程。
- 两个进程通过启动命令临时注入不同环境变量实现隔离（`CHAIN_ENV`、`PORT`）。
- 脚本注入的是“进程级参数”，不会改写你磁盘上的 `.env` 文件内容。
- 因此可同时服务 `sepolia` 和 `local`，且数据仍保持隔离。

## 14. 独立 verifier 与 PDF 验真
当前合同策略已经定死：
1. 合同正文继续存数据库
2. 合同正文不上 IPFS
3. 服务器自带合同验真直接看数据库
4. 用户独立验真依赖本地保存的合同 PDF

当前导出的合同 PDF 内嵌以下验证标记：
- `VERIFY_CONTRACT_ID=<contractId>`
- `VERIFY_CONTENT_HASH=<contentHash>`
- `VERIFY_TX_HASH=<txHash>`
- `VERIFY_LISTING_ID=<listingId>`
- `VERIFY_TENANT_SIGNER=<tenantWallet>`
- `VERIFY_LANDLORD_SIGNER=<landlordWallet>`
- `VERIFY_TENANT_MESSAGE_HASH=<tenantMessageHash>`
- `VERIFY_LANDLORD_MESSAGE_HASH=<landlordMessageHash>`
- `VERIFY_CONTENT_JSON_B64_001...`
- `VERIFY_TENANT_MESSAGE_B64_001...`
- `VERIFY_LANDLORD_MESSAGE_B64_001...`
- `VERIFY_TENANT_SIGNATURE_B64_001...`
- `VERIFY_LANDLORD_SIGNATURE_B64_001...`

当前独立 verifier 对合同 PDF 的校验方式是强校验：
1. 从 PDF 文本层提取完整合同 JSON、完整签名消息原文、完整签名值
2. 本地重算合同 `contentHash`
3. 本地重算租客/房东消息哈希
4. 使用签名值恢复签名地址并自验签
5. 再与链上合同记录、链上消息哈希和房源快照锚点比对
6. 结合链上合同状态与时间字段，派生：
   - 首笔付款是否已记录
   - 合同当前是否已生效
   - 合同是否已过期
   - 合同时间线（统一按 `Asia/Shanghai` / 中国时间显示）

说明：
- Base64 仅用于机器稳定读取，不是加密
- 旧 PDF 兼容已取消；若缺少强校验材料，需要重新下载新的合同 PDF
- 当前 PDF 版式已拆成四段：
  1. 合同正文
  2. 验真摘要
  3. 验真材料明文区
  4. 机器验真标记附录

合同独立验真脚本：
- `verifier/scripts/verify-contract-pdf.js`

独立验真前端启动脚本：
- `scripts/ps1/start-verifier.ps1`
- `scripts/mac/start-verifier.sh`

独立验真前端默认地址：
- `http://127.0.0.1:3010`
- Windows 启动脚本会在启动前自动结束旧的 `3010` 端口 verifier 进程，再重新拉起新实例

合同验真结果页当前会显示：
- 付款状态
- 当前生效状态
- 合同时间线（中国时间）
- 支付总额
- 平台手续费
- 房东实收

## 15. 平台手续费（当前实现）
- 当前仅对首笔租金支付收取平台手续费。
- 费率固定为 `0.1%`，即 `10 bps / 10000`。
- 手续费从房东应收中扣除，租客支付总额不变。
- 例如首笔支付 `0.01 ETH` 时：
- 平台手续费：`0.00001 ETH`
- 房东实收：`0.00999 ETH`
- 前端合同页、合同 PDF、平台验真接口与独立 verifier 都会显示这三个值：
- 支付总额
- 平台手续费
- 房东实收


## 16. 启动脚本选型（更新）
推荐：
1. Sepolia 单网络：`scripts/ps1/start-sepolia-services.ps1` 或 `scripts/mac/start-sepolia-services.sh`
2. Local 单网络：`scripts/ps1/start-local-services.ps1` 或 `scripts/mac/start-local-services.sh`
3. 并行双网络：`scripts/ps1/start-parallel-services.ps1` 或 `scripts/mac/start-parallel-services.sh`
4. Windows 本地 IPFS：`scripts/ps1/setup-local-ipfs.ps1` + `scripts/ps1/start-local-ipfs.ps1`
5. macOS 本地 IPFS：`scripts/mac/setup-local-ipfs.sh` + `scripts/mac/start-local-ipfs.sh`
6. 独立验真前端：`scripts/ps1/start-verifier.ps1` / `scripts/mac/start-verifier.sh`

可直接右键运行：
- 可以在资源管理器中直接右键运行上述 `.ps1` 文件。
- 脚本会自动清理旧端口占用后，再弹出对应服务终端窗口。
- Sepolia 单网络会清理 `3005/3000/3001`；Local 单网络会清理 `3005/3002/3001`；并行模式会清理 `3005/3000/3002/3001`。
- 关闭对应服务终端窗口即可停止对应服务进程。

说明：
- `scripts/start-basic-services.ps1` 已移除，统一使用上述 3 个明确脚本。
- 独立 verifier 不依赖平台前端；如需脱离平台账号系统做房源 / 合同核验，应单独启动 `start-verifier`。
- 独立 verifier 不再提供单独的 PowerShell / shell 命令行包装脚本；正式入口是本地 Web 前端。
