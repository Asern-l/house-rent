# 配置与部署教程

## 1. 环境要求

- Node.js 18 或更高版本。
- MetaMask 或其他 EVM 钱包。
- Windows PowerShell 或 macOS shell。
- Sepolia 测试网部署需要可用 RPC 和测试 ETH。
- Local 环境使用 Ganache，本地链 ID 为 `31337`。

## 2. 安装依赖
在项目根目录执行：

```bash
npm --prefix apps/backend install
npm --prefix apps/frontend install
npm --prefix blockchain install
cd verifier && npm install
```

## 3. 后端环境变量
后端配置文件为 `apps/backend/.env`，可参考 `apps/backend/.env.example`。

```env
JWT_SECRET=change_me_to_a_long_random_string
PORT=3000
HOST=127.0.0.1
AUTH_PORT=3005
CHAIN_ENV=sepolia
JSON_BODY_LIMIT=100mb
PAYMENT_WINDOW_HOURS=2

# DeepSeek 智能搜索
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx

# RPC 覆盖项
SEPOLIA_RPC_URL=https://ethereum-sepolia.publicnode.com
LOCAL_RPC_URL=http://127.0.0.1:8545

# IPFS
IPFS_ENABLED=1
IPFS_API_URL=http://127.0.0.1:5001/api/v0
IPFS_GATEWAY_URL=http://127.0.0.1:8080/ipfs
```

字段说明：

| 字段 | 说明 |
|---|---|
| `JWT_SECRET` | JWT 签名密钥。 |
| `PORT` | 当前业务后端端口。 |
| `HOST` | 服务监听地址，默认 `127.0.0.1`。 |
| `AUTH_PORT` | 独立认证服务端口，默认 `3005`。 |
| `CHAIN_ENV` | 当前后端网络，取值 `sepolia` 或 `local`。 |
| `JSON_BODY_LIMIT` | JSON 请求体上限，用于图片等大体积内容。 |
| `PAYMENT_WINDOW_HOURS` | 首笔支付窗口小时数。 |
| `DEEPSEEK_API_KEY` | 智能搜索 API Key；未配置时使用本地正则解析。 |
| `SEPOLIA_RPC_URL` | Sepolia RPC。 |
| `LOCAL_RPC_URL` | 本地链 RPC。 |
| `IPFS_ENABLED` | 是否启用 IPFS 材料上传。 |
| `IPFS_API_URL` | IPFS API 地址。 |
| `IPFS_GATEWAY_URL` | IPFS Gateway 地址。 |

启动脚本会按网络注入 `CHAIN_ENV` 和端口，不需要手动频繁修改 `.env`。

## 4. 前端环境变量
前端配置文件为 `apps/frontend/.env`，可参考 `apps/frontend/.env.example`。

```env
VITE_DEFAULT_NETWORK=sepolia
VITE_API_BASE_AUTH=/api-auth
VITE_API_BASE_SEPOLIA=/api
VITE_API_BASE_LOCAL=/api-local
VITE_CONTRACT_ADDRESS_SEPOLIA=0xYourSepoliaContractAddress
VITE_CONTRACT_ADDRESS_LOCAL=0xYourLocalContractAddress
```

字段说明：

| 字段 | 说明 |
|---|---|
| `VITE_DEFAULT_NETWORK` | 默认网络。 |
| `VITE_API_BASE_AUTH` | 认证服务代理前缀。 |
| `VITE_API_BASE_SEPOLIA` | Sepolia 业务 API 代理前缀。 |
| `VITE_API_BASE_LOCAL` | Local 业务 API 代理前缀。 |
| `VITE_CONTRACT_ADDRESS_SEPOLIA` | Sepolia 合约地址。 |
| `VITE_CONTRACT_ADDRESS_LOCAL` | Local 合约地址。 |

## 5. 区块链部署变量
`blockchain/.env` 用于合约部署。

```env
SEPOLIA_RPC_URL=https://ethereum-sepolia.publicnode.com
PRIVATE_KEY=0xYourDeployPrivateKey
PAYMENT_WINDOW_HOURS=2
TRUSTED_SIGNER_ADDRESS=0xTrustedSignerAddress
PLATFORM_FEE_RECIPIENT_ADDRESS=0xPlatformFeeRecipientAddress
```

说明：

- `PRIVATE_KEY` 只用于部署脚本。
- `TRUSTED_SIGNER_ADDRESS` 是平台 permit 签名地址。
- `PLATFORM_FEE_RECIPIENT_ADDRESS` 是平台手续费接收地址。
- `PAYMENT_WINDOW_HOURS` 会写入合约构造参数。

## 6. Sepolia 部署

```bash
npm --prefix blockchain run compile
npm --prefix blockchain run deploy:sepolia
npm run sync:abi
```

部署结果写入：

- `blockchain/deployments-rental-sepolia.json`
- `apps/frontend/src/shared/blockchain/RentalChainABI.json`
- `apps/frontend/.env` 中的 Sepolia 合约地址
- `verifier/config/runtime.json` 中的合约配置

启动 Sepolia 服务：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-sepolia-services.ps1
```

## 7. Local 部署
启动本地链：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-persistent-local-node.ps1
```

部署本地合约：

```bash
npm --prefix blockchain run compile
npm --prefix blockchain run deploy:local
npm run sync:abi
```

启动 Local 服务：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-local-services.ps1
```

Local 环境说明：

- RPC：`http://127.0.0.1:8545`
- Chain ID：`31337`
- 默认测试账户余额：`1000 ETH`
- 外部钱包余额低于 `1 ETH` 时，后端会补足到 `50 ETH`

写入 Local 演示数据：

```bash
node scripts/seed-local-listings.js
node scripts/seed-local-listings.js --clear
```

该脚本使用本地测试账号写入演示用户和北京房源数据，仅作用于 Local 数据库。

## 8. 双环境并行运行
并行启动：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-parallel-services.ps1
```

端口分配：

| 服务 | 地址 |
|---|---|
| 认证服务 | `http://127.0.0.1:3005` |
| Sepolia 后端 | `http://127.0.0.1:3000` |
| Local 后端 | `http://127.0.0.1:3002` |
| 前端 | `http://127.0.0.1:3001` |

前端代理：

- `/api-auth` -> 认证服务。
- `/api` -> Sepolia 后端。
- `/api-local` -> Local 后端。

## 9. MetaMask 配置

### 9.1 Sepolia
在 MetaMask 中启用测试网络并选择 Sepolia。

### 9.2 Local EVM
手动添加网络：

| 字段 | 值 |
|---|---|
| 网络名称 | `Local EVM (31337)` |
| RPC URL | `http://127.0.0.1:8545` |
| Chain ID | `31337` |
| 货币符号 | `ETH` |

本地测试私钥以 Ganache 启动终端输出为准。

## 10. IPFS 本地节点
IPFS 用于房源图片、房源公开快照、房源反馈正文和租后评价正文。

Windows：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/setup-local-ipfs.ps1
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-local-ipfs.ps1
```

macOS：

```bash
bash scripts/mac/setup-local-ipfs.sh
bash scripts/mac/start-local-ipfs.sh
```

默认端口：

| 服务 | 地址 |
|---|---|
| API | `http://127.0.0.1:5001` |
| Gateway | `http://127.0.0.1:8080` |
| Swarm | `4001` |

验证：

```powershell
Invoke-WebRequest http://127.0.0.1:5001/api/v0/version -Method Post
```

## 11. 独立验真工具
启动：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-verifier.ps1
```

访问：

```text
http://127.0.0.1:3010
```

- 房源验真。
- 房源详情查询。
- 合约配置保存和导入。
- 应急托管月租释放。

## 12. 重置与重部署
Sepolia：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/reset-sepolia-and-redeploy.ps1
```

Local：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/reset-local-and-redeploy.ps1
```

重置脚本会处理数据库、合约部署、前端合约地址和 ABI 同步。
