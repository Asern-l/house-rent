# 配置与部署教程

## 文档目的
- 指导项目从本地运行到合约部署的完整配置流程。
- 重点说明：环境变量、前后端启动、合约部署、地址回填。

## 一、环境准备
1. 安装 Node.js 18+。
2. 准备 MetaMask 钱包。
3. 准备 Sepolia 测试网 ETH（用于部署与交易 gas）。

## 二、安装依赖
在项目根目录 `CCL_housing` 下执行：

```bash
npm --prefix apps/backend install
npm --prefix apps/frontend install
npm --prefix blockchain install
```

## 三、后端配置
1. 复制示例文件：
- `apps/backend/.env.example` -> `apps/backend/.env`

2. 配置项说明：
- `PORT`：后端端口，默认 `3000`
- `JWT_SECRET`：JWT 密钥（生产环境必须替换）

示例：
```env
PORT=3000
JWT_SECRET=replace_with_your_secret
```

3. 端口联动说明（原方案）
- 当前项目采用“前端代理写死目标地址”的方案。
- 当你修改后端端口时，需要**同步修改两处**：
- 第一处：`apps/backend/.env` 中的 `PORT`
- 第二处：`apps/frontend/vite.config.js` 中 `server.proxy['/api'].target` 的端口
- 若两处不一致，会出现登录/注册请求超时（如 `vite http proxy error: ETIMEDOUT`）。

4. 数据迁移说明（支付模型调整）
- 当前版本已移除“预付款/续费”业务链路，统一为一次性首笔支付（`initial`）。
- 旧数据库在后端启动迁移时会自动收敛为当前模型（仅保留 `initial` 支付类型）。

## 四、前端配置
前端通过 Vite 环境变量读取合约地址。  
在 `apps/frontend` 下新建 `.env`（如不存在）：

```env
VITE_CONTRACT_ADDRESS=0x你的合约地址
```

说明：
- 若合约地址未更新，前端可能仍指向旧地址，导致链上写入失败或写入到旧合约。

## 五、合约部署教程（Sepolia）

## 5.1 配置部署环境变量
在 `blockchain` 目录下新建 `.env`：

```env
SEPOLIA_RPC_URL=https://ethereum-sepolia.publicnode.com
PRIVATE_KEY=你的部署钱包私钥
```

注意：
- `PRIVATE_KEY` 仅用于部署或需要后端签名的场景。
- 已部署完成后，日常前端使用不需要保留部署私钥。

## 5.2 执行编译与部署
在 `blockchain` 目录执行：

```bash
npx hardhat compile
npx hardhat run scripts/deploy_rental.js --network sepolia
```

部署成功后会输出新合约地址，并生成部署信息文件（如 `deployments-rental-sepolia.json`）。

## 5.3 回填前端合约地址
将新地址写入：
- `apps/frontend/.env` 中的 `VITE_CONTRACT_ADDRESS`

修改后重启前端：
```bash
npm --prefix apps/frontend run dev
```

## 六、启动系统

1. 启动后端：
```bash
npm --prefix apps/backend run dev
```

2. 启动前端：
```bash
npm --prefix apps/frontend run dev
```

3. 或一键启动：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-basic-services.ps1
```

## 七、验证建议
1. 后端健康检查：
- 访问 `http://localhost:3000/api/health`

2. 前端功能验证：
- 登录/注册
- 发布房源
- 租客发起合同
- 双方签署并上链
- 验真页查询合同结果

3. 回归脚本：
```bash
node scripts/test_contract_sign_flow.js
```

## 八、常见问题
1. 提示网络不支持
- 检查 MetaMask 是否切换到 Sepolia。

2. 合约地址错误
- 确认 `VITE_CONTRACT_ADDRESS` 是否为最新部署地址。

3. 部署失败（余额不足）
- 给部署钱包补充 Sepolia 测试币。

4. 签署失败但状态已变
- 查看日志：`logs/sign-flow-error.log`。
5. 房源发布失败
- 查看日志：`logs/listing-error.log`。
