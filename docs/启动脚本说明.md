# 启动脚本说明

## 文档目的
- 统一说明项目内启动、重置、IPFS 与独立验真脚本的用途。
- 按操作系统分开说明，避免 Windows 与 macOS 混用命令。

## Windows PowerShell 脚本

### 1. `scripts/ps1/start-sepolia-services.ps1`
功能：
- 启动认证服务（3005）
- 启动 Sepolia 后端（3000）
- 启动前端（3001）

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-sepolia-services.ps1
```

### 2. `scripts/ps1/start-local-services.ps1`
功能：
- 启动认证服务（3005）
- 启动 Local 后端（3002）
- 启动前端（3001）

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-local-services.ps1
```

说明：
- 需先启动本地链节点（见第 4 项）。

### 3. `scripts/ps1/start-parallel-services.ps1`
功能：
- 同时启动：
  - auth(3005)
  - backend(sepolia:3000)
  - backend(local:3002)
  - frontend(3001)

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-parallel-services.ps1
```

### 4. `scripts/ps1/start-persistent-local-node.ps1`
功能：
- 启动可持久化本地链节点（Ganache）
- 数据目录：`blockchain/.ganache-db`
- 若 `8545` 已有旧 Ganache 进程在跑，会先停止旧进程再重启
- 若存在 `blockchain/.ganache-db/LOCK` stale lock，会先清理再启动
- 默认生成 10 个本地测试账户，每个初始余额 `1000 ETH`

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-persistent-local-node.ps1
```

### 5. `scripts/ps1/reset-sepolia-and-redeploy.ps1`
功能：
- 重置 Sepolia 对应数据库与日志
- 重编译并重新部署 Sepolia 合约
- 回填前端 Sepolia 合约地址并同步 ABI

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/reset-sepolia-and-redeploy.ps1
```

### 6. `scripts/ps1/reset-local-and-redeploy.ps1`
功能：
- 重置 Local 对应数据库与日志
- 重建本地链并重新部署 Local 合约
- 回填前端 Local 合约地址并同步 ABI

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/reset-local-and-redeploy.ps1
```

### 7. `scripts/ps1/setup-local-ipfs.ps1`
功能：
- 下载项目内使用的 Kubo 二进制
- 不要求全局安装 `ipfs`

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/setup-local-ipfs.ps1
```

固定版本：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/setup-local-ipfs.ps1 -Version 0.41.0
```

### 8. `scripts/ps1/start-local-ipfs.ps1`
功能：
- 启动项目内本地 IPFS 节点
- 首次启动自动初始化 `.ipfs-data/`
- 默认开放：
  - API：`127.0.0.1:5001`
  - Gateway：`127.0.0.1:8080`

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-local-ipfs.ps1
```

说明：
- 该脚本是唯一负责启动本地 IPFS 节点的 Windows 入口。
- 业务服务脚本不会自动拉起 IPFS。

### 9. `scripts/ps1/start-verifier.ps1`
功能：
- 启动独立验真工具自己的本地 Web 前端
- 默认地址：`http://127.0.0.1:3010`
- 如果 `3010` 已有旧 verifier 进程在跑，会先停止旧进程再重新启动

执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/ps1/start-verifier.ps1
```

说明：
- 首次使用前需先执行 `cd verifier && npm install`

## 补充说明
- Windows 与 macOS 脚本命名保持对应，但执行方式不同。
- 独立 verifier 不依赖平台前端；如需脱离平台账号系统做房源 / 合同核验，应单独启动 `start-verifier`。
- IPFS 节点与业务服务已解耦，便于单独测试独立验真程序。
