# 启动脚本说明

## 文档目的
- 说明两个常用脚本的用途、参数和执行方式。
- 统一 Sepolia / Local 双环境的启动与重置流程。

## 脚本 1：`scripts/start-sepolia-services.ps1`

### 功能
- 安装后端与前端依赖。
- 拉起 Sepolia 后端和前端开发服务。
- 同时拉起独立认证服务（auth）。

### 示例
```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-sepolia-services.ps1
```

### 实际行为
1. 认证服务以 `AUTH_PORT=3005` 启动。
2. 后端以 `CHAIN_ENV=sepolia`、`PORT=3000` 启动。
3. 前端以 `VITE_DEFAULT_NETWORK=sepolia` 启动。
4. 启动前会自动清理 `3005`、`3000`、`3001` 旧占用进程，避免端口冲突。

## 脚本 2：`scripts/start-local-services.ps1`

### 功能
- 安装后端与前端依赖。
- 拉起 Local 后端和前端开发服务。
- 同时拉起独立认证服务（auth）。

### 示例
```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-local-services.ps1
```

### 实际行为
1. 认证服务以 `AUTH_PORT=3005` 启动。
2. 后端以 `CHAIN_ENV=local`、`PORT=3002` 启动。
3. 前端以 `VITE_DEFAULT_NETWORK=local` 启动。
4. 需先启动本地链节点（见脚本 4）。
5. 启动前会自动清理 `3005`、`3002`、`3001` 旧占用进程，避免端口冲突。

## 脚本 3：重置并重部署脚本（已拆分）

### 功能
- 清理当前环境对应的本地数据库。
- 按环境重新编译并部署合约。
- 自动回填前端环境变量中的对应合约地址并同步 ABI。

### 参数
- 无参数：Sepolia 与 Local 分别使用独立脚本。

### 示例
```powershell
powershell -ExecutionPolicy Bypass -File scripts/reset-sepolia-and-redeploy.ps1
powershell -ExecutionPolicy Bypass -File scripts/reset-local-and-redeploy.ps1
```

### 实际行为
1. 删除 `apps/backend/data/database.sepolia.sqlite 或 apps/backend/data/database.local.sqlite`。
2. 删除 `logs/sign-flow-error.log`。
3. 删除 `blockchain/deployments-rental-*.json` 对应环境文件。
4. 执行：
- `npm run compile`
- `npm run deploy:sepolia` 或 `npm run deploy:local`
5. 回填前端变量：
- Sepolia -> `VITE_CONTRACT_ADDRESS_SEPOLIA`
- Local -> `VITE_CONTRACT_ADDRESS_LOCAL`
6. 同步 ABI 到 `apps/frontend/src/shared/blockchain/RentalChainABI.json`。

## 提醒
- `local` 部署前需先启动本地链节点（如 `npx hardhat node`）。
- `sepolia` 部署需要 `blockchain/.env`（含 `SEPOLIA_RPC_URL`、`PRIVATE_KEY`）。

## 脚本 4：`scripts/start-persistent-local-node.ps1`
### 功能
- 启动可持久化本地链节点（Ganache）。
- 数据目录：`blockchain/.ganache-db`。

## 脚本 5：`scripts/start-parallel-services.ps1`
### 功能
- 同时启动：
  - auth(3005)
  - backend(sepolia:3000)
  - backend(local:3002)
  - frontend(3001)
- 便于在同一个前端页面切换网络进行并行联调。
- 启动前会自动清理 `3005/3000/3002/3001` 旧占用进程，避免重复启动冲突。

## 说明
- 单网络启动已正式拆分为两个独立脚本：
- 只跑 Sepolia：`start-sepolia-services.ps1`
- 只跑 Local：`start-local-services.ps1`
