# 启动脚本说明

## 文档目的
- 说明项目中两个常用脚本的用途、执行方式与适用场景。
- 降低“怎么启动”“怎么重置并重新部署”的沟通成本。

## 脚本 1：`scripts/start-basic-services.ps1`

### 功能
- 一键安装依赖并启动后端、前端开发服务。

### 执行方式
在项目根目录 `CCL_housing` 下执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-basic-services.ps1
```

### 实际行为
1. 进入 `apps/backend` 执行 `npm install`。  
2. 新开 PowerShell 窗口启动后端：`npm run dev`。  
3. 进入 `apps/frontend` 执行 `npm install`。  
4. 新开 PowerShell 窗口启动前端：`npm run dev`。  

### 适用场景
- 日常本地开发启动。
- 演示前快速拉起基础服务。

## 脚本 2：`scripts/reset-data-and-redeploy.ps1`

### 功能
- 一键清理本地数据并重新部署合约，同时自动回填前端合约地址与 ABI。

### 执行方式
在项目根目录 `CCL_housing` 下执行：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/reset-data-and-redeploy.ps1
```

### 前置条件
- `blockchain/.env` 已存在，且包含有效 `SEPOLIA_RPC_URL` 与 `PRIVATE_KEY`。
- 部署钱包具备足够 Sepolia 测试币。

### 实际行为
1. 删除后端本地数据库：`apps/backend/data/database.sqlite`。  
2. 删除签署日志：`logs/sign-flow-error.log`。  
3. 删除旧部署文件：`blockchain/deployments-rental-sepolia.json`。  
4. 执行合约编译与部署：  
   - `npx hardhat compile`  
   - `npx hardhat run scripts/deploy_rental.js --network sepolia`  
5. 从新部署结果读取合约地址，写入 `apps/frontend/.env` 的 `VITE_CONTRACT_ADDRESS`。  
6. 同步合约 ABI 到前端：`apps/frontend/src/shared/blockchain/RentalChainABI.json`。  

### 适用场景
- 你修改了合约并需要重新部署。  
- 需要从“干净数据状态”开始联调。  

## 常见提醒
- `reset-data-and-redeploy.ps1` 会清空本地数据库，执行前确认是否需要备份数据。
- 脚本执行完成后，仍需单独启动前后端（脚本末尾已给出提示命令）。
