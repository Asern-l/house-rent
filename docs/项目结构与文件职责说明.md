# 项目结构与文件职责说明

## 文档目的
- 作为当前仓库的人工维护索引。
- 说明目录职责、核心源码职责、核心文档职责。
- 只登记需要人理解和维护的内容，不登记运行生成物与临时调试文件。

## 更新规则
1. 新增长期维护文件时，必须同步更新本文件。
2. 删除长期维护文件时，必须同步删除本文件中的对应说明。
3. 文件职责变化时，必须同步更新本文件，不允许“文件还在、说明已过时”。
4. 自动生成物、运行产物、临时调试脚本不在本文件登记。

## 不登记范围
以下内容默认不在本文件登记：
- `node_modules/`
- `dist/`
- `logs/` 下运行生成的具体日志文件
- `apps/backend/data/*.sqlite`
- `apps/backend/data/uploads/`
- 根目录与子目录中的 `.tmp_*` 临时调试脚本
- 纯部署产物、缓存文件、锁文件的内部细节

## 1. 根目录

### `apps/`
- 承载前后端应用源码。

### `blockchain/`
- 承载智能合约、部署配置、ABI 来源与部署脚本。

### `docs/`
- 承载人工维护文档。
- 当前文档结构分为：
  - 根文档：面向使用、部署、日志、结构说明
  - `schemes/onchain-redesign/`：上链设计方案
  - `todo-sync/`：未完全落地或后续专题待办

### `scripts/`
- 承载本地开发、重置重部署、测试与数据准备脚本。

### `verifier/`
- 承载独立验真程序。
- 与平台前端、平台后端目录分离维护。
- 负责本地 Web 前端、CLI 脚本、链上读取与 IPFS 材料校验。

### 根文件
- [package.json](/D:/0Project/house-rent/package.json)：根脚本入口，聚合前后端、区块链、同步 ABI 等常用命令。
- [README.md](/D:/0Project/house-rent/README.md)：项目总览与快速启动入口。

## 2. 后端应用 `apps/backend`

### 目录
- `src/`
  - 后端核心源码。
- `data/`
  - 数据库存放目录。
  - 也承载本地上传图片目录。
  - 属于运行数据目录，不逐个登记具体文件。

### 配置与入口
- [apps/backend/package.json](/D:/0Project/house-rent/apps/backend/package.json)：后端依赖与脚本定义。
- [apps/backend/.env.example](/D:/0Project/house-rent/apps/backend/.env.example)：后端环境变量样例。

### 核心源码
- [apps/backend/src/index.js](/D:/0Project/house-rent/apps/backend/src/index.js)
  - 后端启动入口。
  - 注册中间件、路由。
  - 负责定时任务：
    - 合同超时取消
    - 合同到期结束
    - 统一链上操作台账补偿与确认

- [apps/backend/src/db.js](/D:/0Project/house-rent/apps/backend/src/db.js)
  - 业务库连接、建表、迁移、持久化与结果解析。
  - 管理房源、合同、支付、版本、gas 授权、统一链上操作台账等业务表。

- [apps/backend/src/onchain-operations.js](/D:/0Project/house-rent/apps/backend/src/onchain-operations.js)
  - 统一链上操作台账写入辅助模块。
  - 负责 `onchain_operations` 的 pending / confirmed / failed 状态更新与结构化结果落库。

- [apps/backend/src/user-db.js](/D:/0Project/house-rent/apps/backend/src/user-db.js)
  - 共享用户库连接与迁移。
  - 负责账号、钱包绑定、风控计数等共享账户数据。

- [apps/backend/src/auth.js](/D:/0Project/house-rent/apps/backend/src/auth.js)
  - JWT 鉴权与角色校验中间件。

- [apps/backend/src/auth-server.js](/D:/0Project/house-rent/apps/backend/src/auth-server.js)
  - 独立认证服务入口。
  - 与业务后端分离，固定承载认证相关接口。

- [apps/backend/src/logger.js](/D:/0Project/house-rent/apps/backend/src/logger.js)
  - 统一日志写入工具。
  - 负责网络分流、公共日志与结构化日志字段。

- [apps/backend/src/app-error.js](/D:/0Project/house-rent/apps/backend/src/app-error.js)
  - 应用层统一错误响应辅助模块。
  - 负责统一错误对象格式、错误响应发送与可抛出业务错误结构。

### 路由模块
- [apps/backend/src/routes/auth.js](/D:/0Project/house-rent/apps/backend/src/routes/auth.js)
  - 钱包签名登录/注册、资料读取与修改。
  - 在 `local` 网络登录时负责触发本地链自动补款检查。

- [apps/backend/src/routes/listings.js](/D:/0Project/house-rent/apps/backend/src/routes/listings.js)
  - 房源发布、查询、房东管理、上链准备与回写。
  - 当前房源链上动作主路径已切到统一 `onchain_operations`。
  - `listing_chain_operations` 已移除；房源链上操作统一由 `onchain_operations` 承担。
  - 当前还负责把房源公开展示态收口成：
    - `可申请`
    - `签约中`
    - `已租出`
    - `已下架`
    - `已关闭`

- [apps/backend/src/routes/contracts.js](/D:/0Project/house-rent/apps/backend/src/routes/contracts.js)
  - 合同创建、条款协商、租客签署、房东签署并上链、支付回写、取消、续约。
  - 当前已把合同创建、首笔支付、gas 锁仓撤销接入统一 `onchain_operations` 异步确认模型。
  - `contract.create` 运行时主确认也已切到统一补偿任务。
  - 当前是合同主业务状态机的核心路由。

- [apps/backend/src/routes/verify.js](/D:/0Project/house-rent/apps/backend/src/routes/verify.js)
  - 房源验真、合同验真。
  - 负责把数据库快照与链上快照做结构化对比。
  - 房源验真当前会在必要时先按链上权威值自愈数据库快照，再输出比对结果。
