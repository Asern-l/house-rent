# 后端接口与前端调用文档

## 1. 服务地址

| 服务 | 默认地址 | 说明 |
|---|---|---|
| 认证服务 | `http://127.0.0.1:3005` | 登录、注册、用户资料。 |
| Sepolia 后端 | `http://127.0.0.1:3000` | Sepolia 业务接口。 |
| Local 后端 | `http://127.0.0.1:3002` | Local 业务接口。 |
| 前端代理认证 | `/api-auth` | 代理到认证服务。 |
| 前端代理 Sepolia | `/api` | 代理到 Sepolia 后端。 |
| 前端代理 Local | `/api-local` | 代理到 Local 后端。 |

## 2. 响应格式
成功响应一般使用：

```json
{
  "success": true,
  "data": {}
}
```

错误响应使用：

```json
{
  "success": false,
  "error": "错误说明",
  "code": "ERROR_CODE"
}
```

部分验真和独立工具接口使用 `ok/result` 结构，含义与 `success/data` 一致。

## 3. 鉴权方式
需要登录的接口使用 Bearer Token：

```http
Authorization: Bearer <token>
```

前端按当前网络保存登录态。Sepolia 与 Local 登录态互相隔离。

## 4. 认证接口
认证接口挂载在 `/api-auth`。

| 方法 | 路径 | 说明 | 鉴权 |
|---|---|---|---|
| `GET` | `/nonce?walletAddress=<address>` | 获取钱包签名 nonce。 | 否 |
| `POST` | `/login` | 钱包签名登录或注册。 | 否 |
| `GET` | `/me` | 获取当前用户资料。 | 是 |
| `PUT` | `/me` | 更新用户资料。 | 是 |
| `POST` | `/me/avatar` | 更新用户头像。 | 是 |

`POST /login` 主要字段：

```json
{
  "walletAddress": "0x...",
  "signature": "0x...",
  "message": "...",
  "role": "tenant"
}
```

`role` 支持 `tenant` 和 `landlord`。

## 5. 房源接口
房源接口挂载在 `/api/listings` 或 `/api-local/listings`。

| 方法 | 路径 | 说明 | 鉴权 |
|---|---|---|---|
| `POST` | `/upload-images` | 上传房源图片。 | 房东 |
| `POST` | `/prepare-create` | 准备房源创建参数和 permit。 | 房东 |
| `POST` | `/commit-create` | 回写房源创建链上交易。 | 房东 |
| `POST` | `/parse-search` | 解析智能搜索条件。 | 否 |
| `GET` | `/` | 房源列表。 | 否 |
| `GET` | `/my` | 房东自己的房源列表。 | 房东 |
| `GET` | `/:id` | 房源详情。 | 否 |
| `GET` | `/:id/history` | 房源历史版本。 | 否 |
| `POST` | `/:id/status/prepare` | 准备房源状态修改。 | 房东 |
| `POST` | `/:id/status/commit` | 回写房源状态修改交易。 | 房东 |
| `POST` | `/:id/terms/prepare` | 准备房源条款修改。 | 房东 |
| `POST` | `/:id/terms/commit` | 回写房源条款修改交易。 | 房东 |
| `POST` | `/:id/clauses` | 更新房源默认附加条款。 | 房东 |
| `DELETE` | `/:id/images` | 删除房源图片。 | 房东 |
| `GET` | `/:id/logs` | 查看房源操作日志。 | 房东 |
| `POST` | `/:id/feedbacks/prepare-onchain` | 准备房源反馈链上参数。 | 登录用户 |
| `POST` | `/:id/feedbacks` | 回写房源反馈链上交易。 | 登录用户 |

房源列表常用查询参数：

| 参数 | 说明 |
|---|---|
| `keyword` | 关键词，匹配标题、描述和地址。 |
| `district` | 区域筛选。 |
| `minRent` | 最低租金。 |
| `maxRent` | 最高租金。 |
| `bedrooms` | 卧室数量；传 `4` 表示 `4室+`，即四室及以上。 |
| `status` | 房源状态。 |

`POST /parse-search` 返回 `engine` 字段，取值为 `deepseek`、`regex` 或 `none`。解析结果还可包含 `sortBy`，取值为 `rent_asc`、`rent_desc`、`area_desc`、`area_asc` 或 `newest`。前端会展示当前智能搜索使用的解析引擎和排序条件，并基于初始房源列表应用解析后的筛选与排序。后端 `GET /listings` 仍支持通过查询参数直接筛选房源。

| 方法 | 路径 | 说明 | 鉴权 |
|---|---|---|---|
| `POST` | `/` | 租客申请签约。 | 租客 |
| `GET` | `/` | 我的合同列表。 | 是 |

| `GET` | `/:id/versions` | 合同条款版本。 | 合同参与方 |

| `POST` | `/:id/clauses/submit` | 租客提交条款。 | 租客 |
| `POST` | `/:id/clauses/review` | 房东审核条款。 | 房东 |
| `GET` | `/:id/gas-authorization/prepare` | 准备 gas 补偿锁仓参数。 | 租客 |
| `POST` | `/:id/sign-tenant` | 租客签署并回写 gas 锁仓。 | 租客 |
| `POST` | `/:id/gas-authorization/revoke` | 回写 gas 锁仓撤销交易。 | 合同参与方 |
| `POST` | `/:id/create-onchain-permit` | 准备合同上链 permit。 | 房东 |
| `POST` | `/:id/sign-landlord` | 房东签署并回写合同上链交易。 | 房东 |
| `GET` | `/:id/payments/prepare-onchain` | 准备首笔支付链上参数。 | 租客 |
| `POST` | `/:id/payments/onchain` | 回写首笔支付交易。 | 租客 |
| `GET` | `/:id/payments` | 合同支付记录。 | 合同参与方 |
| `POST` | `/:id/cancel` | 未付款前取消合同。 | 合同参与方 |
| `POST` | `/:id/terminate-early/onchain` | 回写提前解约交易。 | 租客 |
| `POST` | `/:id/renewals` | 发起续约合同。 | 租客 |
| `GET` | `/:id/pdf` | 下载合同 PDF。 | 合同参与方 |
| `POST` | `/:id/review/prepare-onchain` | 准备租后评价链上参数。 | 租客 |
| `POST` | `/:id/review/onchain` | 回写租后评价交易。 | 租客 |
| `POST` | `/:id/sign-client-report` | 上报前端签署异常。 | 是 |

合同申请主要字段：

```json
{
  "listingId": "lst_xxx",
  "leaseMonths": 1,
  "startDate": "2026-06-03",
  "additionalClauses": []
}
```

首笔支付回写主要字段：

```json
{
  "txHash": "0x...",
  "payType": "initial",
  "amount": "0.01"
}
```

提前解约回写主要字段：

```json
{
  "txHash": "0x..."
}
```

## 7. 验真接口
验真接口挂载在 `/api/verify` 或 `/api-local/verify`。

| 方法 | 路径 | 说明 | 鉴权 |
|---|---|---|---|
| `GET` | `/listing/:id` | 房源验真。 | 否 |
| `GET` | `/contract/:id` | 合同验真。 | 否 |
| `POST` | `/contract-pdf` | 上传 PDF 做合同验真。 | 否 |

房源验真返回：

- 数据库房源快照。
- 链上房源记录。
- 字段级比对结果。
- 当前合同映射语义。

合同验真返回：

- 合同内容哈希比对。
- 双方签名比对。
- 支付记录比对。
- 链上合同记录。
- 当前生效语义。

## 9. 健康检查

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/health` | Sepolia 后端健康检查。 |
| `GET` | `/api-local/health` | Local 后端健康检查。 |

健康检查返回网络、数据库文件、合约配置和待确认链上操作数量。

## 10. 前端调用约定

- 登录相关请求始终走 `/api-auth`。
- 业务请求根据当前网络走 `/api` 或 `/api-local`。
- 链上交易由前端钱包发起，后端只接收交易哈希并校验链上结果。
- 所有关键回写接口都校验 receipt、目标合约、方法名、调用参数和事件。
- 前端在关键交易前执行 `staticCall` 和 `estimateGas`，用于提前发现不可执行交易。
