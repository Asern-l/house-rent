# 后端接口与前端调用文档

## 基础说明
- 后端基地址：`http://localhost:3000/api`
- 返回格式：
  - 成功：`{ success: true, data?: any, message?: string }`
  - 失败：`{ error: string }`

## 认证模块 `/auth`
1. `POST /auth/register`
- 说明：用户注册。
- 请求体：`email, password, role, walletAddress?, nickname?`
- 前端调用：登录页注册流程。

2. `POST /auth/login`
- 说明：用户登录并获取 JWT。
- 请求体：`email, password`
- 前端调用：登录页。

3. `GET /auth/me`
- 说明：获取当前登录用户资料。
- 请求头：`Authorization: Bearer <token>`
- 前端调用：`AuthContext` 初始化。

4. `PUT /auth/me`
- 说明：更新昵称、钱包地址。钱包采用强绑定策略：首次绑定后不可更改/解绑，仅允许提交同一地址用于重连。
- 请求头：`Authorization`
- 前端调用：个人中心、钱包绑定模块。

## 房源模块 `/listings`
1. `POST /listings/upload-images`
- 说明：房东上传房源图片（可选能力，发布前先调用）。
- 请求头：`Authorization`
- 请求体：`{ images: [{ dataUrl }] }`
- 约束：
- 单次最多 `12` 张。
- 支持 `image/jpeg`、`image/png`、`image/webp`。
- 单张最大 `10MB`。
- 返回：`{ success: true, data: { images: [{ url, hash, size, mime }] } }`
- 前端调用：发布房源页图片上传流程。

2. `POST /listings/prepare-create`
- 说明：预创建房源，返回上链锚点参数（先上链后入库）。
- 请求体关键字段：`title, description, address, rentAmount, imageUrls[]`
- 返回关键字段：`draft`, `chainAnchor(listingId, landlordWallet, contentHash)`
- 请求头：`Authorization`
- 前端调用：发布房源页第一步。

3. `POST /listings/commit-create`
- 说明：钱包上链成功后提交 `txHash`，后端校验锚点并真正入库。
- 请求体关键字段：`draft, chainAnchor, txHash`
- 请求头：`Authorization`
- 前端调用：发布房源页第二步。

4. `POST /listings`
- 说明：旧单步发布接口，当前返回 `410`（已下线，防止绕过上链流程）。
- 请求头：`Authorization`

5. `GET /listings`
- 说明：获取房源列表（支持 `keyword`、`status` 查询）。
- 前端调用：房源列表页。

6. `GET /listings/:id`
- 说明：获取房源详情。
- 前端调用：房源详情页。

7. `GET /listings/my`
- 说明：获取当前房东的房源列表。
- 请求头：`Authorization`
- 前端调用：我的房源页。

8. `PUT /listings/:id/status`
- 说明：旧接口，已下线（返回 `410`）。
- 请求头：`Authorization`
- 前端调用：我的房源页。

9. `POST /listings/:id/status/prepare`
- 说明：状态变更预检查，返回链上调用参数（必须先钱包上链）。
- 请求体：`{ status: available|offline|closed }`
- 请求头：`Authorization`
- 前端调用：我的房源页。

10. `POST /listings/:id/status/commit`
- 说明：钱包上链成功后回写状态。
- 请求体：`{ status, txHash }`
- 请求头：`Authorization`
- 前端调用：我的房源页。

## 合同模块 `/contracts`
1. `POST /contracts`
- 说明：租客发起合同。
- 请求头：`Authorization`
- 请求体：`listingId, startDate(YYYY-MM-DD), leaseMonths(1-12，且不小于房源最少租期)`
- 前端调用：房源详情页“申请签约”。

2. `POST /contracts/:id/sign-tenant`
- 说明：租客签署合同。
- 请求头：`Authorization`
- 前端调用：合同详情页。

3. `POST /contracts/:id/sign-landlord`
- 说明：房东签署并上链；同一前端流程内完成消息签名、链上交易与 `tx_hash` 回写。
- 请求头：`Authorization`
- 前端调用：合同详情页。

4. `POST /contracts/:id/cancel`
- 说明：签署阶段取消合同。
- 请求头：`Authorization`
- 前端调用：合同详情页。

5. `POST /contracts/:id/payments/onchain`
- 说明：支付交易上链后回写支付记录（当前仅支持 `initial`），成功后激活合同（首笔一次性支付）。
- 请求头：`Authorization`
- 请求体：`txHash, amount, payType(initial)`
- 前端调用：合同详情页支付流程。

7. `GET /contracts/:id/payments`
- 说明：查询合同下的支付记录列表。
- 请求头：`Authorization`
- 前端调用：合同详情页支付记录区块。

8. `POST /contracts/:id/sign-client-report`
- 说明：前端将签署异常自动上报到日志。
- 请求头：`Authorization`
- 前端调用：签署失败捕获逻辑。

9. `GET /contracts`
- 说明：获取“我的合同”列表。
- 请求头：`Authorization`
- 前端调用：我的合同页。

10. `GET /contracts/:id`
- 说明：获取合同详情。
- 前端调用：合同详情页。

## 房源端错误上报 `/listings`
1. `POST /listings/client-report`
- 说明：前端将房源流程异常主动上报到后端日志（含阶段、错误消息、栈、页面与网络信息）。
- 请求头：`Authorization`
- 请求体：`listingId, stage, message, stack, preferredNetwork, walletAddress, chainId, pageUrl, extra`
- 前端调用：`MyListings` 页面状态变更、条款编辑、运行时异常捕获逻辑。

## 验真模块 `/verify`
1. `GET /verify/listing/:id`
- 说明：查询房源存证信息。
- 返回关键字段：`listingId, txHash, createdAt`。
- 前端调用：验真页。

2. `GET /verify/contract/:id`
- 说明：查询合同哈希比对结果、链上回写状态，以及首笔一次性支付核验结果（用于判断合同是否已生效）。
- 返回关键字段：`contractId, listingId, hashMatch, txHash, paymentVerified`。
- 前端调用：验真页。
