# 后端接口与前端调用文档

## 基础说明
- 后端基地址：`http://localhost:3000/api`
- 返回格式：
  - 成功：`{ success: true, data?: any, message?: string }`
  - 失败：`{ error: string }`

## 认证模块 `/auth`
1. `POST /auth/register`
- 说明：用户注册。
- 请求体：`phone, password, role, walletAddress?, nickname?`
- 前端调用：登录页注册流程。

2. `POST /auth/login`
- 说明：用户登录并获取 JWT。
- 请求体：`phone, password`
- 前端调用：登录页。

3. `GET /auth/me`
- 说明：获取当前登录用户资料。
- 请求头：`Authorization: Bearer <token>`
- 前端调用：`AuthContext` 初始化。

4. `PUT /auth/me`
- 说明：更新昵称、钱包地址。钱包采用强绑定策略：首次绑定后不可更改/解绑，仅允许提交同一地址用于重连。
- 请求头：`Authorization`
- 前端调用：个人中心、钱包绑定模块。

## 房源模块 `/listings`
1. `POST /listings`
- 说明：房东发布房源。
- 请求头：`Authorization`
- 前端调用：发布房源页。

2. `GET /listings`
- 说明：获取房源列表（支持 `keyword`、`status` 查询）。
- 前端调用：房源列表页。

3. `GET /listings/:id`
- 说明：获取房源详情。
- 前端调用：房源详情页。

4. `GET /listings/my`
- 说明：获取当前房东的房源列表。
- 请求头：`Authorization`
- 前端调用：我的房源页。

5. `PUT /listings/:id/status`
- 说明：房东上下架房源（`available/offline`）。
- 请求头：`Authorization`
- 前端调用：我的房源页。

## 合同模块 `/contracts`
1. `POST /contracts`
- 说明：租客发起合同。
- 请求头：`Authorization`
- 请求体：`listingId, startDate(YYYY-MM-DD), leaseMonths(1-12，且不小于房源最少租期)`
- 前端调用：房源详情页“申请签约”。

2. `POST /contracts/:id/sign-tenant`
- 说明：租客签署合同。
- 请求头：`Authorization`
- 前端调用：合同详情页。

3. `POST /contracts/:id/sign-landlord`
- 说明：房东签署合同，返回 `contentHash` 等上链参数。
- 请求头：`Authorization`
- 前端调用：合同详情页。

4. `POST /contracts/:id/onchain`
- 说明：上链交易完成后回写 `txHash`。
- 请求头：`Authorization`
- 请求体：`txHash`
- 前端调用：合同详情页。

5. `POST /contracts/:id/cancel`
- 说明：签署阶段取消合同。
- 请求头：`Authorization`
- 前端调用：合同详情页。

6. `POST /contracts/:id/payments/onchain`
- 说明：支付交易上链后回写支付记录（当前仅支持 `initial`），成功后激活合同（首笔一次性支付）。
- 请求头：`Authorization`
- 请求体：`txHash, amount, payType(initial)`
- 前端调用：合同详情页支付流程。

7. `GET /contracts/:id/payments`
- 说明：查询合同下的支付记录列表。
- 请求头：`Authorization`
- 前端调用：合同详情页支付记录区块。

8. `POST /contracts/:id/sign-client-report`
- 说明：前端将签署异常自动上报到日志。
- 请求头：`Authorization`
- 前端调用：签署失败捕获逻辑。

9. `GET /contracts`
- 说明：获取“我的合同”列表。
- 请求头：`Authorization`
- 前端调用：我的合同页。

10. `GET /contracts/:id`
- 说明：获取合同详情。
- 前端调用：合同详情页。

## 验真模块 `/verify`
1. `GET /verify/listing/:id`
- 说明：查询房源存证信息。
- 前端调用：验真页。

2. `GET /verify/contract/:id`
- 说明：查询合同哈希比对结果、链上回写状态，以及首笔一次性支付核验结果（用于判断合同是否已生效）。
- 前端调用：验真页。
