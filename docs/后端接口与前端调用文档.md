# 后端接口与前端调用文档

## 1. 基础约定

- 业务后端基础路径：`/api`
- 认证后端基础路径：`/api/auth`
- 成功响应：`{ success: true, data?: any, message?: string }`
- 失败响应：`{ code: string, error: string, ...extra }`
- 错误码文档：[错误码清单.md](/D:/0Project/house-rent/docs/错误码清单.md)
- 统一错误模块：[app-error.js](/D:/0Project/house-rent/apps/backend/src/app-error.js)

## 2. 认证模块 `/api/auth`

### `GET /nonce`
- 说明：获取钱包登录一次性 `nonce`
- 鉴权：否
- 前端调用：[LoginPage.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/LoginPage.jsx)

### `POST /login`
- 说明：钱包签名登录；用户不存在时自动注册
- 鉴权：否
- 请求体：`walletAddress`、`signature`、`message`、`timestamp`、`nonce`、`role`、`nickname?`、`phone?`、`preferredNetwork`
- 关键行为：校验钱包签名；当 `preferredNetwork=local` 时检查本地链余额，低于 `50 ETH` 自动补足到 `50 ETH`
- 前端调用：[AuthContext.jsx](/D:/0Project/house-rent/apps/frontend/src/app/providers/AuthContext.jsx)、[LoginPage.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/LoginPage.jsx)

### `GET /me`
- 说明：获取当前用户信息
- 鉴权：`Authorization: Bearer <token>`
- 前端调用：[AuthContext.jsx](/D:/0Project/house-rent/apps/frontend/src/app/providers/AuthContext.jsx)

### `PUT /me`
- 说明：更新当前用户昵称、手机号等资料
- 鉴权：登录用户

## 3. 房源模块 `/api/listings`

### `POST /upload-images`
- 说明：上传房源图片
- 鉴权：房东
- 请求体：`images: [{ dataUrl }]`
- 约束：最多 `12` 张；仅支持 `jpeg/png/webp`；单张最大 `10MB`
- 返回：`images: [{ url, hash, size, mime }]`
- 前端调用：[PublishListing.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/PublishListing.jsx)、[MyListings.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/MyListings.jsx)

### `POST /client-report`
- 说明：前端主动上报房源流程异常
- 鉴权：登录用户
- 常见字段：`listingId`、`stage`、`message`、`stack`、`preferredNetwork`、`walletAddress`、`chainId`、`pageUrl`、`extra`
- 前端调用：[MyListings.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/MyListings.jsx)

### `POST /prepare-create`
- 说明：生成房源发布草稿与链上锚点
- 鉴权：房东
- 请求体：`title`、`description`、`address`、`district?`、`rentAmount`、`minLeaseMonths`、`clauses[]`、`bedrooms`、`livingrooms`、`bathrooms`、`area`、`imageUrls[]`
- 返回：`draft`、`chainAnchor`
- 前端调用：[PublishListing.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/PublishListing.jsx)

### `POST /commit-create`
- 说明：钱包完成 `createListing(...)` 后回写房源上链结果
- 鉴权：房东
- 请求体：`draft`、`chainAnchor`、`txHash`、`operationId`
- 关键行为：写入 `onchain_operations` 的 `listing.create` 记录，并校验 receipt、目标合约、事件与参数；查不到 receipt 时返回 `202`
- 前端调用：[PublishListing.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/PublishListing.jsx)

### `GET /`
- 说明：获取公开房源列表
- 鉴权：否
- 返回关键字段：房源快照、`public_status`、`active_contract_id`、`signing_contract_id`
- 前端调用：[ListingsPage.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/ListingsPage.jsx)

### `GET /my`
- 说明：获取当前房东的房源列表
- 鉴权：房东
- 返回关键字段：房源快照、`public_status`
- 前端调用：[MyListings.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/MyListings.jsx)

### `GET /:id`
- 说明：获取房源详情
- 鉴权：否
- 前端调用：[ListingDetail.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/ListingDetail.jsx)

### `GET /:id/history`

- 鉴权：否

### `POST /:id/status/prepare`
- 说明：修改房源状态前的链上预检查
- 鉴权：房东
- 请求体：`status: available | offline | closed`
- 返回：`listingId`、`fromStatus`、`toStatus`、`toStatusEnum`
- 前端调用：[MyListings.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/MyListings.jsx)

### `POST /:id/status/commit`
- 说明：钱包完成 `setListingStatus(...)` 后回写
- 鉴权：房东
- 请求体：`status`、`txHash`、`operationId?`

### `POST /:id/terms/prepare`
- 说明：修改租金、最少租期、图片前的链上预检查
- 鉴权：房东
- 请求体：`rentAmount`、`minLeaseMonths`、`imageUrls[]`、`clauses[]?`
- 返回：`contentHash`、`chainAnchor`
- `chainAnchor` 关键字段：`contentHash`、`snapshotCid`、`snapshotHash`

### `POST /:id/terms/commit`
- 说明：钱包完成 `updateListingTerms(...)` 后回写
- 鉴权：房东
- 请求体：`rentAmount`、`minLeaseMonths`、`imageUrls[]`、`clauses[]`、`txHash`、`chainAnchor`、`operationId?`
- 关键行为：写入 `onchain_operations` 的 `listing.terms` 记录，并校验 `ListingContentUpdated`；查不到 receipt 时返回 `202`
- 前端调用：[MyListings.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/MyListings.jsx)

### `POST /:id/clauses`
- 说明：更新房源默认附加条款模板
- 鉴权：房东
- 请求体：`clauses[]`
- 关键行为：仅更新 `listings.clauses_template_json`，不调用钱包，不写链上状态
- 前端调用：[MyListings.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/MyListings.jsx)

### `DELETE /:id/images`
- 说明：删除房源图片并同步图片哈希
- 鉴权：房东
- 请求体：`url`

### `GET /:id/logs`
- 说明：查看房源操作日志
- 鉴权：房东

### `POST /:id/feedbacks/prepare-onchain`
- 说明：房源反馈上链前，先将评论明文写入 IPFS 并生成 `commentCid`
- 鉴权：登录用户，且不能是该房源房东

- 返回：`listingId`、`feedbackType`、`commentText`、`commentHash`、`commentCid`
- 前端调用：[ListingReviewsPage.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/ListingReviewsPage.jsx)

### `POST /:id/feedbacks`
- 说明：钱包完成 `submitListingFeedback(...)` 后回写

- 请求体：`feedbackType`、`commentText`、`commentHash`、`commentCid`、`txHash`

## 5. 验真模块 `/api/verify`

### `GET /listing/:id`
- 说明：房源链上验真
- 鉴权：否
- 关键约束：纯只读接口；只返回数据库快照、链上快照和字段比对结果，不再在公开验真请求中自愈或回写数据库
- 返回关键字段：数据库快照、链上快照、字段比对结果、占用语义、`chainHeadContractId`、`currentEffectiveContractId`、`dbSnapshot.onchainState`
- 前端调用：[VerifyPage.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/VerifyPage.jsx)

### `GET /contract/:id`
- 说明：合同链上验真
- 鉴权：否
- 返回关键字段：哈希比对结果、签名验真结果、支付校验结果、`onchainState`
- 前端调用：[VerifyPage.jsx](/D:/0Project/house-rent/apps/frontend/src/pages/VerifyPage.jsx)
