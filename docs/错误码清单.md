# 错误码清单

## 文档目的
- 统一维护后端错误码。
- 约束错误响应格式。
- 给前端、联调、审查提供稳定参考。

## 公共定义入口
- [app-error.js](D:/0Project/house-rent/apps/backend/src/app-error.js)

当前公共函数：
- `error(code, message, extra)`
- `sendError(res, status, code, message, extra)`
- `fail(status, code, message, extra)`
- `handleAppError(res, err, fallbackStatus, fallbackCode, fallbackMessage)`

## 统一响应格式
所有标准业务错误统一返回：

```json
{
  "code": "ERROR_CODE",
  "error": "中文描述",
  "...extra": "可选附加字段"
}
```

## 命名规则
- 全大写
- 下划线分隔
- 前缀按业务语义，不强制按模块名

示例：
- `CONTRACT_NOT_FOUND`
- `TENANT_ONLY`
- `TX_HASH_INVALID`
- `PAYMENT_AMOUNT_MISMATCH`

## 使用规则
1. 可预期业务错误
- 优先使用 `sendError(...)`

2. 深层校验/工具函数
- 可使用 `fail(...)`
- 在路由层统一 `handleAppError(...)`

3. 不要再直接返回：
- `{ error: '...' }`
