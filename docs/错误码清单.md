# 错误码清单

## 1. 错误响应格式

```json
{
  "success": false,
  "error": "错误说明",
  "code": "ERROR_CODE"
}
```

字段说明：

| 字段 | 说明 |
|---|---|
| `success` | 固定为 `false`。 |
| `error` | 面向用户或调用方的错误说明。 |
| `code` | 稳定错误码。 |
| 其他字段 | 部分接口会附带上下文，例如当前状态、截止时间或链上状态。 |

## 10. 使用建议

- 前端展示优先使用 `error` 字段。
- 自动化测试和异常分类使用 `code` 字段。
- 链上交易类错误应同时展示交易哈希、当前网络和合约地址，便于排查。
