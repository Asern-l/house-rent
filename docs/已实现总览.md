## B. 房源业务
- 房东发布房源、房源列表、房源详情、我的房源。
- 房源本体状态机：`available`、`offline`、`closed`。
- 房源公开状态三态：`签约中`、`已租出`、`可申请`。
- 房源公开状态按合同时间与支付阶段计算；`pending/tenant_signed` 仅在未超过 `expires_at` 时继续算“签约中”。
- 房东可编辑房源信息：租金、最少租期、图片，并回写链上。
- 房东可编辑默认附加条款模板；该操作只更新数据库，不触发钱包签名和链上写入。
- 房源发布采用“后端生成草稿 + 钱包上链 + 后端回写”流程。

## C. 合同与支付主链路
- 合同创建时生成 `content_json` 与 `content_hash`。
- 新合同申请前，后端统一按房源公开状态判断是否允许申请。
- 合同签署主状态：`pending -> tenant_signed -> pending_payment -> active`。
- 条款协商采用“租客提议、房东处理”模型。
- 租客可在签署前反复保存草稿、提交提议。
- 房东仅在存在租客提议时处理，可直接确认、修改后确认、退回并备注。
- 房东确认更新当前可签署版本。
- 首次签署发生后，协商立即冻结。
- 租客签署同时沉淀合同签名证据与 gas 锁仓记录。
- 租客签署会校验链上 `lockGasCompensationEscrow(...)` 交易、实付 `msg.value`、调用参数与 `GasCompEscrowLocked` 事件后，才推进到 `tenant_signed`。
- gas 授权上限 `capWei` 由后端 `prepare` 固定生成，租客正式签署时必须与后端准备值完全一致。
- 房东签署与合同上链合并为一个前端流程。
- 房东签署回写校验链上 receipt、目标合约、`createContractRecord(...)`、`ContractCreated`、`ContractSignatureAnchored`。
- 支付回写校验链上 receipt、`recordInitialRentPayment(...)`、付款钱包、房东地址、`msg.value`、`RentPaymentRecorded`。
- 租客签署写库采用显式事务：只有合同状态更新、gas 授权写入与 prepare 清理同时成功，才会提交为 `tenant_signed`。
- 链上操作统一进入 `onchain_operations` 台账。
- 统一台账状态：`pending`、`confirmed`、`failed`。
- 房源与合同主表的链上补偿状态统一由 `onchain_operations` 承担。
- 房源与合同详情接口统一返回：`onchain_state`、`onchain_error_message`、`onchain_tx_hash`。
- 合同链上字段包含：`parentContractId`、`renewalChildContractId`、`initialAmountWei`、`startAtMs`、`endAtMs`、`tenantMessageHash`、`landlordMessageHash`、`tenantSignedAt`、`landlordSignedAt`。
- 合同支付窗口由合约构造参数 `paymentWindowMs` 控制。
- 首笔支付由租客钱包直接发起；支付成功后普通合同进入 `active`，续约合同在起租日前显示为待接续状态。
- `activeContractByListing` 表示租赁链头合同索引。
- 合约查询口径区分：`getContractChainHeadByListing` 返回合同链头，`getCurrentEffectiveContractByListing` 返回当前按时间仍有效的合同。
- `status == Active` 表示合同曾激活成功；当前是否有效占用仍需结合 `startAtMs/endAtMs` 判断。
- 续约入口已落地：租客可在父合同页发起续约，默认继承父合同最终条款，缺失时使用房源默认附加条款。
- 续约合同默认起始时间等于父合同结束时间，仅允许编辑续约月数。
- 支付前前端执行 `staticCall + estimateGas` 预检查，并对常见 RPC/钱包错误做中文提示。
- gas 补偿采用链上锁仓与即时结算。
- 租客签署前会强校验链上 `lockGasCompensationEscrow(...)` 交易、实付 `msg.value`、调用参数与 `GasCompEscrowLocked` 事件，拒绝伪造锁仓哈希。
- 未生效合同取消与 gas 锁仓退款分离处理：`/contracts/:id/cancel` 只取消合同，不直接伪造 gas 已退款状态；链上退款确认后才回写 `contract_gas_authorizations.status = revoked`。
- 当前若直接取消时 gas 锁仓仍为链上 `active`，后端会明确返回“gas 待释放”，前端继续保留 `取回 gas 预付` 入口。
- 签约超时自动过期只推进合同数据库状态；若 gas 锁仓仍在链上 active，合同参与方可继续在已结束合同页发起撤销授权退款。
- `contract_gas_authorizations` 保存 `lock_tx_hash`、`revoke_tx_hash`、`settle_tx_hash`。
- 合同到期释放当前由数据库自动任务和链上懒释放共同承担；`completeExpiredContract(...)` 已存在于合约 ABI 中，但业务入口尚未接入。
- 合同详情接口仅租客或房东本人可查看。
- 自动任务包括：支付超时取消、合同到期结束、签约超时过期。
- 合同签约有效期：`expires_at = 创建时间 + 24 小时`。

## C1. 签约风控治理

## D. 验真能力
- 房源验真接口。
- 合同验真接口。
- 房源验真返回数据库快照、链上快照、字段比对结果与占用语义。
- 合同验真返回哈希比对、签名验真、支付验真、链上锚定与时间语义。
- 懒释放模式下，合同验真以 `startAtMs/endAtMs` 推导“当前是否仍有效”为准；数据库 `ended` 与链上 `Active` 在时间语义一致时不视为失败。
- 独立验真程序已拆出到 `verifier/`，不再与平台前端/后端目录共用入口。
- 独立验真支持：
- 本地合同 PDF 验真
- 房源链上验真
- 房源公开快照 + IPFS 验真
- 合同联动房源验真
- 房源详情查看
- 房源反馈与租后评价的 `commentCid` 联动验真
- 可选查询房源全部快照历史版本
- 合同 PDF 强校验：
  - 本地重算合同 `contentHash`
  - 本地重算租客/房东消息哈希
  - 使用签名值恢复地址并自验签
- 合同 PDF 排版已拆为：
  - 正文页
  - 验真摘要页
  - 验真材料明文区
  - 机器验真标记附录

## D1. 评价系统
- 房源反馈：公开展示、链上锚定 `feedbackType + commentHash + commentCid`、链下存储评论明文，不计入正式星级。
- 房源反馈按钱包地址公开展示，允许重复提交。
- 租后评价：单向，仅真实租客可提交。
- 租后评价公开展示钱包地址、星级和评论，链上锚定 `rating + commentHash + commentCid`。
- 租后评价按租期月数加权；至少 `3` 条后才公开平均分。
- 房源详情页展示评价摘要；完整评论区位于独立房源评论页。
- 房源详情接口返回：
- `feedbacks`
- `tenant_reviews`
- `review_summary`
- 房源反馈回写接口：
- `POST /listings/:id/feedbacks`
- 合同详情接口返回：
- `rental_review`
- `review_state`
- 房源独立验真会联动读取链上评论事件，按 `commentCid` 从 IPFS 拉取正文并校验 `commentHash`。

关键位置：

## G. 区块链与部署
- 合约工程与部署脚本可用（Sepolia / localhost）。
- 合约已接入 OpenZeppelin `ECDSA` 与 `ReentrancyGuard`。
- 首笔支付仅允许合同租客本人发起；支付函数按 Checks-Effects-Interactions 顺序更新状态后再转账。
- 房源链上关键字段修改（条款、状态）在存在阻塞合同链时会被合约直接拒绝。
- 房源链上条款修改与状态修改入口同样受 `nonReentrant` 保护。
- 前端 ABI 同步流程已固化在重置重部署脚本中。
