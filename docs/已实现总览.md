## B. 房源业务
- 房东发布房源、房源列表、房源详情、我的房源。
- 房源状态机：`available`、`locked`、`rented`、`offline`、`closed`。
- 状态管理约束：支持 `available <-> offline`，并支持房东手动销毁为 `closed`（终态，不可恢复）。
- 房源公开状态已收口为前端三态：`签约中`、`已租出`、`可申请`；判断优先依据合同时间与支付阶段，而不是单纯依赖数据库 `listings.status`。
- 房源详情默认展示房东邮箱；租客申请签约面板展示租客邮箱，便于双方线下协商。
- 房源发布为后端入库流程；房源链上交易需由钱包侧独立发起（不再后端代发）。

关键位置：

## C. 合同与支付主链路
- 合同创建：生成 `content_json` 与 `content_hash`。
- 并发保护：创建时原子锁房源（`available -> locked`）。
- 双方签署流转：`pending -> tenant_signed -> pending_payment`。
- 租客签署现在同时包含两类证据：
- 合同签署消息签名（用于 `tenantMessageHash` / PDF 留档）。
- gas 补偿锁仓签名与链上锁仓交易（`lockGasCompensationEscrow(...)`）。
- 房东签署与合同上链已合并：房东点击“房东签署并上链”后，在同一前端流程内完成消息签名、链上交易与后端 `tx_hash` 回写，不再保留独立“合同上链”按钮。
- 合同链上字段已扩展：
- `parentContractId`
- `renewalChildContractId`
- `initialAmountWei`
- `startAtMs`
- `endAtMs`
- `tenantMessageHash`
- `landlordMessageHash`
- `tenantSignedAt`
- `landlordSignedAt`
- 合同链上全局支付窗口改为 `paymentWindowMs`：部署时由 `PAYMENT_WINDOW_HOURS` 换算传入构造函数，链上直接校验首笔支付是否超时。
- 首笔一次性支付由租客钱包直接发起链上转账；回写成功后，普通合同进入 `active`，续约合同若起租时间尚未到达则进入“已支付待接续”的展示态。
- 合同链上已引入 `ContractStatus` 与 `activeContractByListing`：`activeContractByListing` 现在表达“这条租赁链的链头合同”，不再表达“当前生效合同”。
- 已明确约束：`status == Active` 只表示“合同曾经被激活/支付成功过”，不直接等于“当前仍有效占用”；当前占用仍需结合 `startAtMs/endAtMs` 判断。
- 已落地续约入口：仅租客可在父合同页发起续约，续约合同默认继承父合同最终条款；若父合同缺少条款，则回退房东房源默认条款。
- 续约合同默认起始时间固定为父合同 `endDate`，仅允许编辑续约月数。
- 已移除后端 `payments/authorization` 平台代签链路，不再要求配置支付授权私钥。
- 支付前增加钱包会话与 RPC 预检查：前端会先执行 `staticCall + estimateGas`，并将 `-32002 / could not coalesce error / missing revert data` 归类为中文提示。
- gas 补偿闭环已改为链上即时结算：租客签署时先链上锁仓 `lockGasCompensationEscrow(...)`，房东点击“房东签署并上链”时合约按公式 `estimatedGasUnits * tx.gasprice * 3` 当场报销并把剩余退回租客。
- 未生效合同取消已补链上统一释放函数：房东或租客取消未生效合同时，前端统一调用 `cancelPendingGasAuthorization(...)`，若锁仓仍为 `active`，合约会当场退款给租客并改为 `revoked`；后端仅负责回写 `revoke_tx_hash` 与数据库状态。
- `contract_gas_authorizations` 持久化链上关键交易哈希：`lock_tx_hash / revoke_tx_hash / settle_tx_hash`；其中 `settle_tx_hash` 与房东签署并上链这笔交易哈希一致。
- 合同到期释放已支持链上显式函数 `completeExpiredContract(...)`，并在合同创建/支付时对过期旧合同做懒释放。
- 已新增专题文档 `docs/todo-sync/合同到期释放与懒释放机制说明.md`，用于单独说明显式释放、懒释放与服务器检测分工。
- 合同取消与房源释放逻辑。
- 旧的提前解约后端接口已下线，不再暴露 `/contracts/:id/terminations*` 这组历史残留路由。
- 上链回写幂等约束：`tx_hash` 仅允许首次写入。
- 支付回写幂等约束：同交易哈希不可重复入账。
- 客户端失败上报接口。
- 合同详情页增加哈希算法展示：`SHA-256`（用于 `content_hash` 计算说明）。
- 定时任务：
- 房东签署后按 `PAYMENT_WINDOW_HOURS` 未支付自动取消。
- 租期到期自动结束并释放房源。
- `pending/tenant_signed` 到 `expires_at` 自动过期（`expired`）并释放房源（无其他进行中合同时）。
- 合同签约有效期：`expires_at = 创建时间 + 24 小时`（首次签约与修订合同均一致）。
