# 已实现总览（统一维护）

## 文档目的
- 作为唯一“已实现能力”总览文档长期维护。
- 与 `docs/待实现总览.md` 配套，形成闭环。

## A. 账号与鉴权
- 用户注册/登录（`landlord` / `tenant`）。
- JWT 鉴权与角色权限校验。
- 个人信息查询与更新。
- 钱包强绑定策略：首次绑定后不可改绑，仅支持重连同一地址。
- 独立认证服务（auth service，默认 `3005`）上线，认证接口与业务后端解耦。

关键位置：
- `apps/backend/src/routes/auth.js`
- `apps/backend/src/auth.js`
- `apps/backend/src/auth-server.js`
- `apps/frontend/src/app/providers/AuthContext.jsx`

## B. 房源业务
- 房东发布房源、房源列表、房源详情、我的房源。
- 房源状态机：`available`、`locked`、`rented`、`offline`、`closed`。
- 上下架约束：仅 `available <-> offline`，流程中状态禁止手动改动。

关键位置：
- `apps/backend/src/routes/listings.js`
- `apps/frontend/src/pages/ListingsPage.jsx`
- `apps/frontend/src/pages/ListingDetail.jsx`
- `apps/frontend/src/pages/MyListings.jsx`
- `apps/frontend/src/pages/PublishListing.jsx`

## C. 合同与支付主链路
- 合同创建：生成 `content_json` 与 `content_hash`。
- 并发保护：创建时原子锁房源（`available -> locked`）。
- 双方签署流转：`pending -> tenant_signed -> pending_payment`。
- 首笔一次性支付（`initial`）回写成功后：`pending_payment -> active`，房源 `locked -> rented`。
- 合同取消与房源释放逻辑。
- 上链回写幂等约束：`tx_hash` 仅允许首次写入。
- 支付回写幂等约束：同交易哈希不可重复入账。
- 客户端失败上报接口。
- 定时任务：
- 房东签署后 2 小时未支付自动取消。
- 租期到期自动结束并释放房源。

关键位置：
- `apps/backend/src/routes/contracts.js`
- `apps/backend/src/index.js`
- `apps/frontend/src/pages/ContractPage.jsx`

## D. 验真能力

关键位置：
