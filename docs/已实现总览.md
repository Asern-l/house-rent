# 已实现功能总览

## 1. 账号与认证

- 支持房东和租客注册、登录。
- 支持钱包地址绑定和签名登录。
- 支持用户资料和头像更新。
- 认证服务独立运行，默认端口 `3005`。
- Sepolia 与 Local 登录态按网络隔离。
- Local 环境支持外部钱包自动补足测试余额。

## 2. 房源管理

- 房东发布房源。
- 房东管理自己的房源。
- 房源支持图片上传、租金、地址、最短租期和附加条款。
- 房源支持上架、下架和关闭。
- 房源关键内容上链存证。
- 房源支持版本化公开快照。
- 房源列表支持智能搜索和区域筛选。
- 智能搜索优先使用 DeepSeek API，未配置时使用本地正则解析。
- 前端会展示智能搜索使用的解析引擎，并用解析结果请求服务端筛选。
- 房源列表 `4室+` 筛选按四室及以上匹配。
- 房源公开状态支持 `可申请`、`签约中`、`已租出`、`已下架`、`已关闭`。

## 3. 合同流程

- 租客从房源详情页发起签约申请。
- 合同生成 `content_json` 和 `content_hash`。
- 支持租客和房东进行条款确认。
- 支持租客签署和 gas 补偿锁仓。
- 支持房东签署并将合同关键事实上链。
- 支持合同 PDF 下载。
- 支持首笔租金链上支付。
- 支持合同支付超时取消。
- 支持合同签署超时处理。
- 支持租客发起续约。
- 支持续约合同支付后待接续展示。
- 支持租客提前解约。
- 父合同提前解约时，尚未生效的续约子合同自动取消。

## 4. 支付与资金规则

- 首笔租金由租客钱包支付到合约。
- 平台手续费为 `0.1%`。
- 履约保证金为房东税费后应收金额的 `10%`，支付时直接给房东。
- 剩余 `90%` 作为托管租金保留在合约中。
- 托管租金按履约时间释放给房东。
- 独立验真工具提供房东应急释放入口。
- 提前解约时，合约按已履约时间结算房东应得托管金额，剩余未履约托管金额退回租客。

## 5. 链上操作台账

- 房源上链、合同上链、支付、评价、反馈等关键链上操作进入统一台账。
- 台账状态包括 `pending`、`confirmed`、`failed`。
- 后端通过 receipt、方法名、参数和事件校验链上交易。
- 前端展示链上状态、交易哈希和错误信息。

## 6. 评价与反馈

- 房源反馈面向房源本身，不计入正式星级。
- 租后评价只允许真实租客提交。
- 租后评价支持星级和评论。
- 提前解约合同支持租后评价。
- 评论正文保存到 IPFS，链上保存哈希和 CID。
- 房源详情页展示评价摘要和评论入口。

## 7. 验真能力

- 平台前端提供房源验真和合同验真。
- 独立验真工具提供合同 PDF 验真、房源验真和房源详情查询。
- 合同 PDF 内嵌合同正文、签名消息、签名值、链 ID、合约地址和机器验真标记。
- 合同验真会重算合同哈希、消息哈希并恢复签名地址。
- 房源验真会比对数据库快照、链上记录和 IPFS 公开材料。
- 独立验真工具支持保存多个合约配置。
- 独立验真工具支持从 PDF 或蓝色验真参数块导入合约配置。

## 8. 通知中心

- 支持站内通知列表。
- 支持未读数展示。
- 支持单条已读和全部已读。
- 通知覆盖合同申请、条款确认、签署、支付、超时、提前解约、评价和反馈等业务事件。

## 9. 环境隔离

- Sepolia 和 Local 使用独立业务数据库。
- Sepolia 和 Local 使用独立账号数据库。
- 前端按当前网络选择不同 API 前缀和合约地址。
- 独立 verifier 使用独立配置，不依赖平台登录态。

## 10. 本地演示能力

- Windows 和 macOS 都提供服务启动脚本。
- Local 环境使用 Ganache 持久化本地链。
- Local 默认测试账户余额为 `1000 ETH`。
- 本地 IPFS 节点可通过脚本安装和启动。
- 独立验真工具可通过脚本启动本地 Web 页面。
- Local 演示用户和北京房源数据可通过 `scripts/seed-local-listings.js` 写入。
