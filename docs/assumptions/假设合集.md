# 假设合集（建模前提）

## 1. 文档定位
- 本文件用于集中维护项目中的“建模假设/工作前提”。
- 假设可被代码、流程、风控、合规、测试共同引用。
- 目标是统一语义、减少歧义、方便后续增删与版本化追踪。

## 2. 使用规则
- 新增假设：在“假设清单”追加一条，并更新版本记录。
- 修改假设：保留原编号，更新状态与变更说明。
- 失效假设：状态改为“失效”，不得直接删除历史条目。

## 3. 假设状态定义
- `生效`：当前默认成立，可直接用于设计与评审。
- `待验证`：暂用前提，需后续证据确认。
- `失效`：不再成立，仅保留历史追溯。

## 4. 假设清单

### H1 身份与钱包绑定假设
- 状态：`生效`
- 假设内容：租客与房东的钱包地址已通过外部实名/KYC体系绑定到对应自然人。
- 适用范围：合同签署归责、链上支付归责、争议主体识别。
- 边界说明：不覆盖匿名开放网络下的地址推断；不自动等于司法已认定。
- 依赖证据：KYC记录、绑定时间、可出示证明材料。
- 影响模块：合同签署、风控、验真、争议处理。
- 最近更新：2026-05-24
