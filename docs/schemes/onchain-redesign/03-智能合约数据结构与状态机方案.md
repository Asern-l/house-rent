# 智能合约数据结构与状态机方案

## 文档范围
- 合约文件：`blockchain/contracts/RentalChain.sol`
- 说明内容：枚举、结构体、状态变量、事件、函数、权限、状态机与当前边界

## 当前阶段实现范围
- 当前项目以“系统原型可跑通”为第一目标，合约当前主要承载三部分：
1. 房源存证与状态流转
2. 合同存证、支付生效、续约接续与房源占用链
3. gas 补偿锁仓、撤销、取消释放与即时结算
- 押金托管、仲裁、完整提前解约链上化当前未启用。

## 1. 设计目标
- 关键状态迁移由合约强约束，避免绕过网站直接改出非法状态。
- 权限、版本、支付金额、支付时效、续约链关系尽量由链上校验。
- 房源本体状态与“是否当前被租/被占位”分开建模。

## 2. 角色与权限
- `landlordWallet`
  - 创建房源
  - 更新房源可改字段
  - 切换房源 `Active / Offline / Closed`
  - 创建合同上链
- `tenantWallet`
  - 锁定 gas 补偿
  - 发起支付
  - 发起续约申请对应的后续签署/支付动作
- `landlordWallet` 与 `tenantWallet`
  - 都可取消未生效合同并触发链上 gas 锁仓释放
- 无常驻平台 `operator`
  - 当前版本不再依赖平台私钥做日常状态推进

## 3. 枚举定义

### 3.1 `ListingStatus`
- `Active`
- `Offline`
- `Closed`

说明：
- 房源链上状态只表示房源本体是否可见、是否可编辑、是否已销毁。
- 房源是否“当前租出”不由 `ListingStatus` 表达。

### 3.2 `ContractStatus`
- `None`
- `Created`
- `Paid`
- `Active`
- `Completed`
- `Cancelled`

说明：
- `Created`：合同已创建/已上链，但未支付
- `Paid`：续约合同已提前支付，但尚未到接续生效时点
- `Active`：合同曾被成功激活/支付成功过
- `Completed`：合同已正式结项
- `Cancelled`：合同已取消

重点说明：
- `status == Active` 不等于“这份合同当前仍然有效占用房源”。
- `status == Active` 只表示“这份合同曾经被激活过/支付成功过”。
- 当前是否仍有效占用，必须继续结合 `startAtMs / endAtMs` 判断。

### 3.3 `GasAuthStatus`
- `None`
- `Active`
- `Revoked`
- `Settled`

说明：
- `Active`：租客资金已锁进合约
- `Revoked`：已撤销/已释放并退款
- `Settled`：已在房东创建合同时参与结算

说明：

### 4.2 `ContractRecord`
字段：
- `string contractId`
- `string listingId`
- `string parentContractId`
- `string renewalChildContractId`

说明：
- `parentContractId` 只用于续约。
- `renewalChildContractId` 只允许一个，用于从父合同向下查唯一续约链。

### 4.3 `GasAuthorization`

## 5. 状态变量与映射
- `mapping(string => ListingRecord) private _listings`
- `string[] private _allListingIds`
- `mapping(string => ContractRecord) private _contracts`
- `mapping(string => string) private _activeContractByListing`
- `mapping(bytes32 => GasAuthorization) private _gasAuths`
- `uint256 public immutable paymentWindowMs`
- `uint256 public constant GAS_REIMBURSE_ESTIMATED_UNITS`
- `uint256 public constant GAS_REIMBURSE_MULTIPLIER`

说明：
- `_activeContractByListing[listingId]` 现语义不是“当前有效合同”，而是“这条租赁链的链头合同”。
- 当前有效合同需要从链头顺着 `renewalChildContractId` 向下结合时间判断。

## 7. 核心状态机

### 7.1 普通合同
1. 房东创建上链 -> `Created`
2. 租客支付成功 -> `Active`
3. 到期或显式结项 -> `Completed`
4. 未生效前取消 -> `Cancelled`

### 7.2 续约合同
1. 仅租客可在父合同页发起
2. 子合同创建上链 -> `Created`
3. 若父合同尚未失效，提前支付 -> `Paid`
4. 父合同失效后，子合同进入接续 -> `Active`
5. 到期或显式结项 -> `Completed`
6. 未生效前取消 -> `Cancelled`

重点：
- 续约允许提前支付
- 提前支付不立刻接管占用
- 一个父合同同一时间只允许一个子合同
- 不允许续约分叉

## 8. 房源占用与当前有效合同判定

### 8.1 房源本体状态
- 只看 `ListingStatus`

### 8.2 当前有效合同
从链头合同开始，顺着 `renewalChildContractId` 往下查。

判定规则：
- `Cancelled / Completed`：不算当前有效
- `Created`：不算当前有效；若仍在支付窗口内，可算“签约中占位”
- `Paid`：若父合同仍未结束，不算当前有效；若父合同已结束，可视为待接续合同
- `Active`：只有 `now < endAtMs` 时才算当前有效

### 8.3 前端房源三态
- `签约中`
  - 存在链下签署流程中的合同
  - 或存在 `Created` 且未过支付窗口的合同
  - 或存在 `Paid` 子合同而父合同尚未结束
- `已租出`
  - 存在 `Active` 且 `now < endAtMs` 的当前有效合同
- `可申请`
  - 排除以上两类，且房源本体状态为 `available`

## 9. 核心函数职责

### 9.1 房源
- `createListing(...)`
- `updateListingTerms(...)`
- `setListingStatus(...)`
- `getListing(...)`
- `getListingCount()`

### 9.2 gas 锁仓
- `lockGasCompensationEscrow(...)`
- `revokeGasCompensationAuthorization(...)`
- `cancelPendingGasAuthorization(contractId, tenant, nonce)`

说明：
- `cancelPendingGasAuthorization(...)` 用于“未生效合同取消时统一释放 gas 锁仓”
- 房东或租客均可触发

### 9.3 合同创建与上链
- `createContractRecord(CreateContractParams params, bytes tenantSignature, bytes landlordSignature)`

行为：
- 创建合同
- 校验续约父子关系
- 写入签署锚点
- 发签名证据事件
- 即时结算 gas 补偿

### 9.4 支付
- `recordInitialRentPayment(contractId, landlord, orderNo)` `payable`

规则：
- 普通合同支付成功 -> `Active`
- 续约合同：
  - 父合同未结束时 -> `Paid`
  - 父合同已结束时 -> `Active`
- 链上使用全局 `paymentWindowMs` 校验支付截止时效
- 校验 `msg.value == initialAmountWei`

### 9.5 到期结项
- `completeExpiredContract(contractId)`

说明：
- 任意地址可调用
- 仅当合同已到期时允许
- 用于显式结项
- 懒清理可以只清索引，不强制改变所有历史状态判断口径

## 10. 支付时效方案
- 不再为每份合同单独存 `paymentDeadlineMs`
- 改为合约级全局参数 `paymentWindowMs`
- 部署时由 `PAYMENT_WINDOW_HOURS` 换算后传入构造函数
- 对任一已部署合约而言，这个窗口固定

## 11. 当前未实现或未完全收口的部分
- 完整提前解约链上状态机
- 提前解约与续约冲突规则
- 押金托管
- 仲裁/争议处理
- 完整的签名事件验真页
- `MyListings.jsx` / 定时任务 / 数据库状态机与链上状态机的进一步收口

## 12. 当前结论
- 房源状态不再承载“是否已租出”的真相。
- 合同链、时间字段与支付阶段共同决定当前是否有效占用。
- `status == Active` 只表示“曾经激活过”，不是“当前一定仍有效”。
- 签名采用：
  - `messageHash` 上链 storage
  - `signature` 通过 event 公开
- 续约采用：
  - `parentContractId`
  - `renewalChildContractId`
  - 单父合同单子合同
  - 允许提前支付，但不立即接管
