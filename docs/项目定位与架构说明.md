# 项目定位与架构说明

## 1. 项目定位
链上安居是一个面向租房业务的链上可信交易系统。系统采用“平台业务层 + 链上可信层 + 独立验真层”的混合架构：

- 平台业务层负责账号、房源、合同、搜索、通知、风控、默认展示和业务编排。
- 链上可信层负责房源关键状态、合同关键事实、签名锚点、支付托管、提前解约、租后评价和房源反馈的不可篡改记录。
- 独立验真层负责在不依赖平台账号系统的情况下，通过合同 PDF、链上合约配置和房源公开材料完成核验。

系统的核心目标是：让租房业务保持可用、可治理，同时让关键合同和交易事实具备可验证性。

## 2. 架构分层

### 2.1 前端应用
前端位于 `apps/frontend`，提供以下能力：

- 用户登录、资料维护和网络切换。
- 房源浏览、智能搜索、区域筛选和房源详情展示。
- 房东发布、编辑和管理房源。
- 租客申请签约、条款确认、签署、支付、续约、提前解约和租后评价。
- 钱包连接、链上交易发起、交易预检查和中文错误提示。
- 平台验真页面，用于查看房源、合同、支付和链上锚点结果。

### 2.2 后端服务
后端位于 `apps/backend`，提供以下能力：

- 独立认证服务和业务后端服务。
- 用户、房源、合同、支付、评价、反馈、通知和日志管理。
- 房源与合同业务状态机。
- 交易许可签发、链上交易回写校验和异步补偿确认。
- IPFS 材料上传和 CID 索引。
- Local 与 Sepolia 双环境数据隔离。
- DeepSeek 智能搜索解析，未配置 API Key 时使用本地正则解析兜底。

### 2.3 智能合约
合约位于 `blockchain/contracts/RentalChain.sol`，负责以下链上规则：

- 房源关键字段存证。
- 合同内容哈希、签名哈希、签署时间和租期锚定。
- 租客 gas 补偿锁仓。
- 首笔租金支付、平台手续费、履约保证金和托管金额拆分。
- 托管月租按履约时间释放。
- 租客提前解约并按规则结算。
- 续约合同链和房源当前合同映射。
- 租后评价与房源反馈哈希锚定。

### 2.4 独立验真工具
独立验真工具位于 `verifier`，提供本地 Web 页面和底层脚本：

- 合同 PDF 验真。
- 房源验真。
- 房源详情查询。
- 合约配置保存、导入和切换。
- 房东应急托管月租释放。

独立验真工具只读取用户提供的 PDF、保存的合约配置、链上数据和公开材料，不依赖平台登录态。

## 3. 数据与可信边界

### 3.1 平台数据库
平台数据库保存业务运行所需数据，包括账号、房源、合同、支付记录、通知、日志索引、IPFS CID 和链上操作台账。数据库用于产品默认展示和业务编排。

### 3.2 链上记录
链上记录保存关键事实，包括房源关键状态、合同哈希、签名锚点、支付托管、提前解约、释放记录、评价和反馈哈希。链上记录用于验真和约束关键业务行为。

### 3.3 IPFS 材料
IPFS 保存公开验真材料，包括房源图片、房源公开快照、房源反馈正文和租后评价正文。合同正文和合同 PDF 不上传 IPFS。

### 3.4 合同 PDF
合同 PDF 是合同独立验真的核心载体。PDF 内包含合同正文、签名材料、链上合约配置和机器验真标记。用户保存 PDF 后，可以通过独立验真工具复核合同与链上记录。

## 4. 环境模型
系统支持两个运行环境：

- `sepolia`：公开测试网环境。
- `local`：本地 Ganache EVM 环境，链 ID 为 `31337`。

两个环境使用独立业务库、独立账号库、独立部署文件和独立合约地址。前端通过网络选择决定请求 `/api` 或 `/api-local`。

## 5. 关键业务口径

### 5.1 房源公开状态
房源对外展示状态由房源本体状态和合同状态共同决定：

- `可申请`：房源可见且没有阻塞合同。
- `签约中`：存在未完成签署或待支付合同。
- `已租出`：存在已支付且当前有效的合同。
- `已下架`：房东主动下架。
- `已关闭`：房源已关闭。

### 5.2 合同业务状态
平台合同状态包括：

- `pending`：待签署。
- `tenant_signed`：租客已签署。
- `pending_payment`：双方签署完成，等待首笔支付。
- `active`：已支付并生效。
- `ended`：租期自然结束。
- `cancelled_before_payment`：未付款前取消。
- `expired`：签署或支付窗口超时。
- `terminated_early`：租客提前解约。

### 5.3 链上合同状态
链上合约状态由 `RentalChain` 管理，包括 `Created`、`Paid`、`Active`、`Completed`、`Cancelled`。平台会结合链上状态、租期时间和数据库状态派生用户可读语义。

## 6. 项目名称
项目中文名称为“链上安居”。英文名称为“Onchain Housing”。
