## 日志目录
- 根目录：`logs/`
- 公共目录：`logs/common/`
- 分网络目录：
- `logs/networks/local/`
- `logs/networks/sepolia/`

## 当前日志文件

### 公共日志
- `logs/common/api-error.log`
  - API 级错误、4xx/5xx 请求结果、中间件异常
- `logs/common/user-event.log`
  - 账号登录、注册、本地补款等用户事件

### 按网络分流日志
- `logs/networks/<network>/sign-flow-error.log`
  - 合同签署、支付、前端主动上报的签约流错误
- `logs/networks/<network>/listing-error.log`
  - 房源上链、房源状态修改、房源前端主动上报错误
- `logs/networks/<network>/risk-event.log`
  - 风控命中、冷却、自动释放等风险事件
- `logs/networks/<network>/system-error.log`
  - 进程级异常，如 `unhandledRejection`、`uncaughtException`

`<network>` 目前取值：
- `local`
- `sepolia`

## 非日志型状态台账
- 数据库表：`apps/backend/data/database.<network>.sqlite` 中的 `onchain_operations`
- 用途：记录已接入链路的链上操作确认状态，不直接写入 `logs/` 目录
- 当前状态值：
  - `pending`
  - `confirmed`
  - `failed`
- 当前已接入操作类型：
  - `listing.create`
  - `listing.status`
  - `listing.terms`
  - `listing.reconcile`
  - `contract.create`
  - `payment.initial`
  - `gas_auth.revoke`
- 关键字段：
  - `op_id`
  - `entity_type`
  - `entity_id`
  - `operation_kind`
  - `tx_hash`
  - `status`
  - `result_json`
  - `error_message`
- 说明：`onchain_operations` 是链上补偿、异步确认与排障的主状态源；结构化日志仍用于记录异常上下文与运行时错误。

## 当前日志函数
- `logApiError(stage, detail)`
  - 写 `logs/common/api-error.log`
- `logUserEvent(stage, detail)`
  - 写 `logs/common/user-event.log`
- `logSignFlow(stage, detail)`
  - 写 `logs/networks/<network>/sign-flow-error.log`
- `logListingError(stage, detail)`
  - 写 `logs/networks/<network>/listing-error.log`
- `logRiskEvent(stage, detail)`
  - 写 `logs/networks/<network>/risk-event.log`
- `logSystemError(stage, detail)`
  - 写 `logs/networks/<network>/system-error.log`

## 网络分流规则
- 优先读取 `detail.preferredNetwork`
- 其次读取 `detail.chainId`
- 再回退到后端当前 `CHAIN_ENV`

目的：
- 前端主动上报时，尽量按用户实际目标网络落日志
- 后端本地环境异常时，仍能按当前运行网络落日志

## 当前后端自动记录

### 请求级自动记录
- 文件：
- [index.js](D:/0Project/house-rent/apps/backend/src/index.js)

已实现：
- 404 统一记 `api-error`
- 500 中间件异常统一记 `api-error`
- 请求完成后自动记录 4xx/5xx
- 请求体过大（413）专项记录

### 链上补偿自动记录
- 文件：
- [index.js](D:/0Project/house-rent/apps/backend/src/index.js)
- [onchain-operations.js](D:/0Project/house-rent/apps/backend/src/onchain-operations.js)

已实现：
- 后台定时扫描 `onchain_operations.status = pending`
- 按 `receipt + calldata + event` 校验链上结果
- 成功时更新台账为 `confirmed`
- 校验失败或交易回执失败时更新台账为 `failed`
- 房源链路只写统一台账 `onchain_operations`，不再保留旧链上操作镜像表

### 进程级自动记录
- `unhandledRejection`
- `uncaughtException`

写入：
- `system-error.log`

- `message`
- `contractId`
