## 已插入日志的位置
1. 后端 `apps/backend/src/routes/contracts.js`
- `POST /contracts/:id/sign-landlord` 的异常分支。
- `POST /contracts/:id/sign-client-report` 接口（前端主动上报）。

1.0 前端 `apps/frontend/src/pages/ContractPage.jsx`
- 合同签署失败主动上报到 `POST /contracts/:id/sign-client-report`。
- 首笔支付失败主动上报到 `POST /contracts/:id/sign-client-report`。
- 上报字段包含：`type`、`preferredNetwork`、`message`、`apiStatus`、`apiError`、`walletAddress`、`chainId`、`pageUrl`，支付额外包含 `contractAddress` 与 `amount`。
- 支付失败日志额外包含 `phase`：用于区分 `payment_precheck`（预检查失败）和 `payment_send`（正式发送失败）。
- 租客 gas 锁仓失败也会主动上报到 `POST /contracts/:id/sign-client-report`，`phase = gas_auth_lock`。

1.1 后端 `apps/backend/src/routes/listings.js`
- `POST /listings/client-report` 接口（前端房源页面主动上报）。
- 覆盖阶段：`status.prepare/start/done`、`status.onchain.send/mined`、`status.error`、`terms.*`、`runtime.window.error`、`runtime.unhandledrejection`。

2. 后端 `apps/backend/src/logger.js`
- `logSignFlow(stage, detail)`：统一写 JSON 行日志。
- `logApiError(stage, detail)`：记录路由 404、中间件异常与 API 级错误。
- `logSystemError(stage, detail)`：记录进程级异常（`unhandledRejection` / `uncaughtException`）。
- `logRiskEvent(stage, detail)`：记录签约风控事件。
- `logUserEvent(stage, detail)`：记录共享账号相关用户事件（公共目录）。
- 自动按 `preferredNetwork/chainId/CHAIN_ENV` 解析网络并分文件写入。

3. 后端 `apps/backend/src/index.js`
- 404 兜底统一写入 `api-error`。
- 500 中间件统一写入 `api-error`（包含 requestId、method、url、stack）。
- 请求完成时自动记录 4xx/5xx 到 `api-error`（包含状态码与耗时）。
- 请求体超限（413）专项记录 `middleware.payload-too-large`，返回“请求体过大”明确提示。
- 进程级异常统一写入 `system-error`。

## 建议继续加日志的位置
1. 钱包切链失败分支（前端合同页）
- 记录 `chainId`、钱包地址、目标网络。

2. 上链交易发送前后（前端合同页）
- 记录 `contractId`、`txHash`、钱包地址。

3. 数据库写入关键点（后端合同路由）
- 合同创建成功、状态迁移成功、上链回写成功。

## 日志格式建议
每行一条 JSON，字段建议至少包含：
- `at`：ISO 时间
- `cnAt`：北京时间（`Asia/Shanghai`）展示字段
- `stage`：阶段标识
- `contractId`
- `userId`
- `requestId`
- `message`

## 本次“上链失败”根因示例
- 日志显示 `eth_sendTransaction` 的 `to` 与 `from` 相同，且都等于钱包地址。
- 这通常表示前端读取到的 `VITE_CONTRACT_ADDRESS_<NETWORK>` 配置值不是合约地址（被误填成钱包地址，或读取到了错误配置）。
