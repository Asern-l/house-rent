# 日志埋点说明

## 1. 日志目录
日志文件位于项目根目录 `logs/`。

| 目录 | 说明 |
|---|---|
| `logs/common/` | 跨网络共享日志，例如认证与用户事件。 |
| `logs/sepolia/` | Sepolia 后端日志。 |
| `logs/local/` | Local 后端日志。 |

运行日志不提交到 Git。

## 2. 日志格式
日志采用 JSON Lines 格式，每行一条 JSON 记录。

通用字段：

| 字段 | 说明 |
|---|---|
| `time` | 日志写入时间。 |
| `event` | 事件名称。 |
| `network` | 网络标识。 |
| `requestId` | 请求 ID。 |
| `userId` | 当前用户 ID。 |
| `role` | 当前用户角色。 |
| `method` | HTTP 方法。 |
| `path` | 请求路径。 |
| `status` | 响应状态码。 |
| `message` | 错误或事件说明。 |
| `details` | 结构化上下文。 |

## 3. 主要日志文件

| 文件 | 说明 |
|---|---|
| `api-error.log` | API 4xx/5xx 错误。 |
| `system-error.log` | 未捕获异常、后台任务异常、系统级错误。 |
| `risk-event.log` | 风控、链上回写、业务关键事件。 |
| `sign-flow.log` | 合同签署流程关键节点。 |
| `listing-error.log` | 房源上链和房源管理异常。 |
| `user-event.log` | 登录、用户资料、头像、Local 自动补款等用户事件。 |

## 4. API 错误日志
API 错误日志记录接口层异常，包括：

- 参数校验失败。
- 鉴权失败。
- 业务状态不允许操作。
- 链上交易校验失败。
- IPFS 上传失败。
- 服务器内部错误。

典型排查步骤：

1. 根据前端错误码定位请求路径。
2. 在对应网络目录查找 `api-error.log`。
3. 使用 `requestId` 关联同一次请求的其他日志。
4. 对链上问题继续检查 `onchain_operations` 台账和交易哈希。

## 5. 系统错误日志
系统错误日志记录后台任务和进程级异常，包括：

- `unhandledRejection`
- `uncaughtException`
- 链上操作补偿任务异常
- 自动过期任务异常
- 数据库保存异常

系统错误优先排查运行环境、RPC、数据库文件和端口占用。

## 6. 风控日志
风控日志记录申请签约限流、冷却期和关键业务风险事件。

合同申请风控包括：

- 同租客同房源 24 小时申请次数限制。
- 同租客全局申请频率限制。
- 冷却期阻断。

常见事件：

| 事件 | 说明 |
|---|---|
| `contract.apply.blocked.active-cooldown` | 当前账号处于冷却期。 |
| `contract.apply.blocked.global-rate` | 全局申请频率超限。 |
| `contract.apply.blocked.same-listing` | 同一房源申请次数超限。 |
| `listing.onchain.commit.create` | 房源创建链上回写。 |
| `listing.onchain.commit.status` | 房源状态链上回写。 |
| `listing.onchain.commit.terms` | 房源条款链上回写。 |

## 7. 签署流程日志
签署流程日志用于排查合同签署、gas 锁仓、房东上链和支付阶段的问题。

重点关注：

- 合同 ID。
- 当前用户 ID。
- 当前钱包地址。
- gas 授权 nonce。
- 锁仓交易哈希。
- 合同上链交易哈希。
- 支付交易哈希。
- 链上回写状态。

## 8. 链上操作台账
链上操作主状态存储在数据库表 `onchain_operations`，不是单纯依赖日志。

关键字段：

| 字段 | 说明 |
|---|---|
| `op_id` | 操作 ID。 |
| `entity_type` | 实体类型，如 `listing`、`contract`、`payment`。 |
| `entity_id` | 实体 ID。 |
| `operation_kind` | 操作类型。 |
| `tx_hash` | 链上交易哈希。 |
| `status` | `pending`、`confirmed`、`failed`。 |
| `error_message` | 失败原因。 |
| `payload_json` | 请求上下文。 |
| `result_json` | 确认结果。 |

排查链上回写时，以 `onchain_operations` 为主，日志为辅助。

## 9. 通知日志
通知表 `notifications` 保存站内通知。通知由合同、房源、支付、评价、反馈、提前解约和自动任务产生。

通知排查重点：

- `recipient_id` 是否正确。
- `kind` 是否正确。
- `dedupe_key` 是否避免重复写入。
- `target_type` 和 `target_id` 是否能跳转到正确页面。

## 10. 常用查看命令
PowerShell：

```powershell
Get-Content logs/sepolia/api-error.log -Tail 50
Get-Content logs/local/system-error.log -Tail 50
Get-Content logs/common/user-event.log -Tail 50
```

按关键字搜索：

```powershell
Select-String -Path logs/sepolia/*.log -Pattern "contractId"
Select-String -Path logs/local/*.log -Pattern "txHash"
```

macOS：

```bash
tail -n 50 logs/sepolia/api-error.log
tail -n 50 logs/local/system-error.log
grep -R "contractId" logs/sepolia
```
