# 日志埋点说明（测试用）

## 日志目录
- 运行日志目录：`CCL_housing/logs`
- 签署问题日志文件：`CCL_housing/logs/sign-flow-error.log`
- 按网络拆分日志文件：
- `CCL_housing/logs/sign-flow-error.sepolia.log`
- `CCL_housing/logs/sign-flow-error.local.log`
- `CCL_housing/logs/listing-error.sepolia.log`
- `CCL_housing/logs/listing-error.local.log`

## 已插入日志的位置
1. 后端 `apps/backend/src/routes/contracts.js`
- `POST /contracts/:id/sign-landlord` 的异常分支。
- `POST /contracts/:id/sign-client-report` 接口（前端主动上报）。

2. 后端 `apps/backend/src/logger.js`
- `logSignFlow(stage, detail)`：统一写 JSON 行日志。
- 自动按 `preferredNetwork/chainId/CHAIN_ENV` 解析网络并分文件写入。

## 建议继续加日志的位置
1. 钱包切链失败分支（前端合同页）
- 记录 `chainId`、钱包地址、目标网络。

2. 上链交易发送前后（前端合同页）
- 记录 `contractId`、`txHash`、钱包地址。

3. 数据库写入关键点（后端合同路由）
- 合同创建成功、状态迁移成功、上链回写成功。

## 日志格式建议
每行一条 JSON，字段建议至少包含：
- `at`：ISO 时间
- `cnAt`：北京时间（`Asia/Shanghai`）展示字段
- `stage`：阶段标识
- `contractId`
- `userId`
- `requestId`
- `message`

## 快速查看日志
```powershell
Get-Content .\logs\sign-flow-error.log -Tail 50
Get-Content .\logs\sign-flow-error.sepolia.log -Tail 50
Get-Content .\logs\sign-flow-error.local.log -Tail 50
```

## 本次“上链失败”根因示例
- 日志显示 `eth_sendTransaction` 的 `to` 与 `from` 相同，且都等于钱包地址。
- 这通常表示前端读取到的 `VITE_CONTRACT_ADDRESS_<NETWORK>` 配置值不是合约地址（被误填成钱包地址，或读取到了错误配置）。
