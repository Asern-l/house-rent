## 8. 测试网与本地链环境隔离切换（未实现）
### 目标
- 支持在“Sepolia 测试网 / 本地链（Hardhat）”之间一键切换。
- 切换后房源、合同、支付、验真等全链路数据互相隔离，避免串数据。

### 当前状态
- 前端网络侧当前仅保留 Sepolia。
- 后端数据库当前为单库模式（同一份 `database.sqlite`）。
- 合约地址当前为单环境配置。

### 缺失点
- 缺少统一环境切换开关（前端 + 后端 + 脚本）。
- 缺少按链环境拆分的独立数据库（例如 `database.sepolia.sqlite` / `database.local.sqlite`）。
- 缺少按链环境拆分的合约地址配置与部署文件管理。
- 缺少“房源/合同/支付/验真”跨环境严格隔离校验。

### 建议实现方向
- 通过 `CHAIN_ENV`（如 `sepolia` / `local`）驱动全局环境。
- 数据层按 `CHAIN_ENV` 选择不同数据库文件。
- 前端按 `CHAIN_ENV` 选择对应 `VITE_CONTRACT_ADDRESS` 与网络参数。
- 脚本按环境输出独立部署文件并回填对应前端配置。

### 验收标准
- 切到 `local` 后只能看到本地链对应数据，切回 `sepolia` 不可见本地数据。
- 两套环境可分别发布房源、发起合同、签署、支付、验真，互不影响。
- 重启服务后各环境数据仍保持独立一致。
