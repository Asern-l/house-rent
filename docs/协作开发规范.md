# 协作开发规范（house-rent）

## 1. 项目基线
- 项目名称：`house-rent`
- 项目路径：`D:\0Project\house-rent`
- 技术栈：
  - 前端：React + Vite + Tailwind + ethers
  - 后端：Node.js + Express + sql.js
  - 合约：Solidity + Hardhat
- 目标：持续迭代“房源发布 -> 合同签署 -> 合同哈希上链 -> 验真查询”主流程，并逐步闭环预留功能。

## 2. 目录与模块约定
- `apps/backend`：后端核心逻辑（鉴权、房源、合同、验真、数据层）
- `apps/frontend`：前端页面与业务交互
- `blockchain`：智能合约、Hardhat 配置、部署脚本
- `scripts`：启动、重置、回归脚本
- `docs`：项目文档
- `logs`：运行日志输出目录

## 3. 代码规范
- 新增或修改函数需保留中文函数编号注释。
- 函数注释与函数实现必须在同一文件。
- 核心功能必须可运行并可验证。
- 冗余代码应清理，复杂逻辑可拆分文件。

## 4. 日志规范
- 后端日志统一复用 `apps/backend/src/logger.js`（或后续统一日志入口）。
- 核心函数输入、输出、关键判断、异常分支应记录日志。
- 日志测试点维护在 `docs/日志埋点说明.md`。

## 5. 文档规范（强制）
- 代码变更后必须同步更新 `docs`。
- 文档引用路径统一使用**相对路径**，禁止写绝对路径（如 `D:\...`）。
- 推荐示例：
  - `apps/backend/src/index.js`
  - `apps/frontend/src/app/App.jsx`
  - `docs/待实现总览.md`
- 至少同步维护：
  - `docs/项目结构与文件职责说明.md`
  - `docs/后端接口与前端调用文档.md`
  - `docs/日志埋点说明.md`
  - `docs/已实现总览.md`
  - `docs/待实现总览.md`

## 6. 接口文档模板（沿用）
新增接口时，在 `docs/后端接口与前端调用文档.md` 补齐：
- 接口名称
- 请求路径与方法
- 鉴权要求
- 请求参数（必填/选填、类型、含义）
- 返回结构（成功/失败）
- 示例调用（请求与响应）

## 7. 执行与验收
- 变更必须落盘到仓库文件，不以聊天文本代替提交。
- 变更完成后至少执行一项验证（后端启动、前端构建、回归脚本等）。
- 环境变量变更需同步维护对应 `*.env.example`。
